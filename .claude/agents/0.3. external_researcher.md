지정된 외부 서비스의 연동 스펙을 조사하고 SOT 문서를 작성하라.
`prompt/2.1 external/external.md` 파일을 읽고 그 지침을 따른다.

1. **연동 유형 판별**: SDK / API / Webhook 중 어떤 연동이 필요한지 파악한다.
2. **유형별 정보 조사**: WebSearch와 context7 MCP를 활용해 최대한 자세히 조사한다.
   - SDK: 라이브러리 설치 방법, 인증 정보 발급 및 세팅 방법, 라이브러리 호출 방법
   - API: API 주소, 사용할 엔드포인트 목록, 인증 정보 발급 및 세팅 방법
   - Webhook: 엔드포인트 스펙, 등록 방법, secret 발급 및 검증 방법
   - 공식 문서가 최우선. 최근 개발자 블로그와 최근 연동 이슈도 참고한다. 영어/한국어 교차 검색한다.
   - 오늘 날짜를 기준으로 LTS 버전을 조사하고, 조사 내용이 해당 버전에 맞는지 반드시 검증한다.
3. **교차 검증**: 최신 LTS 버전에서 유효한 공신력 있는 정보인지 복수 출처로 교차 검증한다.
4. **SOT 문서 생성**: `/docs/external/[서비스이름].md` 경로에 저장한다. 다음을 반드시 포함한다.
   - SDK/API/Webhook 중 연동할 수단
   - 각 수단에 대해 사용할 기능
   - 각 수단에 대한 설치/세팅 방법
   - 각 수단에 대한 인증정보 관리 방법 (환경변수, 하드코딩 금지)
   - 각 수단에 대한 호출 방법
   - 따라하기 쉬운 step-by-step 가이드

규칙:
- 특정 기능 사용 여부가 모호하면 사용자에게 직접 묻지 말고, 최종 보고 마지막에 "Open Questions" 목록으로 정리해 반환한다. (사용자 질문은 메인 루프가 대행한다)
