`docs/techstack.md`(Source of Truth)와 `docs/requirement.md`를 읽고 프로젝트 개발/테스트 환경을 구축하라.

1. **스캐폴딩**: techstack.md의 Codebase Structure에 따라 프로젝트 구조를 생성한다. 공식 스캐폴딩 도구가 있으면 우선 사용한다 (예: `npm create vite@latest`, `django-admin startproject`).
2. **의존성 설치**: techstack.md의 라이브러리 목록을 해당 패키지 매니저로 설치한다.
3. **품질 도구 설정**: 타입 체크, 린트, 포맷터를 구성한다.
4. **환경변수**: `.env.example`을 생성하고 필요한 키를 나열한다. 실제 값과 하드코딩은 금지. `.gitignore`에 `.env`가 포함되어 있는지 확인한다.
5. **테스트 환경 구축**: 단위테스트/E2E테스트 환경을 구성하고, 각각 간단한 예시 테스트를 하나씩 생성한다. (이번 작업의 목표는 환경 구축이므로 예시 테스트는 최소한으로)
6. **검증**: techstack.md에 정의된 빌드/린트/테스트 명령어를 전부 실행하여 모두 통과하는지 확인한다.

최종 보고에 포함할 것:
- 생성된 구조 요약과 설치된 의존성 목록
- 실행한 검증 명령어와 결과 (전부 통과해야 완료)
- 실패한 항목이 있다면 원인과 수정 시도 내역

규칙:
- techstack.md에 없는 기술을 임의로 추가하지 않는다.
- 모든 검증 명령이 통과할 때까지 스스로 수정을 반복한다. 해결 불가능한 항목만 보고서에 남긴다.
