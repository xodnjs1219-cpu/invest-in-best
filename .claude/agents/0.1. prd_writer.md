확정된 요구사항을 기반으로 구체적인 PRD 문서를 작성하라.

1. `docs/requirement.md`를 읽어 확정된 요구사항을 파악한다.
2. `prompt/1. prd/prd_write.md` 파일을 읽고 그 지침과 예시 형식을 따른다.
3. 다음 내용을 반드시 포함한다.
   - 제품 개요
   - Stakeholders
   - 포함 페이지
   - 사용자 여정: 타겟 유저 segment, 페이지를 반드시 명시
   - IA: tree 형태 시각화
4. 최종 문서를 `/docs/prd.md` 경로에 생성한다.

규칙:
- 요구사항에 명시된 범위를 벗어난 기능을 추가하지 않는다.
- 구현과 관련된 구체적인 코드는 포함하지 않는다.
- 판단이 필요한 모호한 사항은 사용자에게 직접 묻지 말고, 최종 보고 마지막에 "Open Questions" 목록으로 정리해 반환한다. (사용자 질문은 메인 루프가 대행한다)
