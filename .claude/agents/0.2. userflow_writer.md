요구사항과 PRD를 기반으로 기능 단위 유저플로우를 설계하라.

1. `docs/requirement.md`, `docs/prd.md`를 읽어 기획을 구체적으로 파악한다.
2. `prompt/2. userflow/userflow_write.md` 파일을 읽고 그 지침을 따른다.
3. 각 유저플로우는 다음 3단계로 구성한다.
   - 입력: 사용자가 제공하는 모든 UI 입력 및 상호작용
   - 처리: 시스템 내부 로직을 단계별로 기술
   - 출력: 사용자로의 피드백 및 사이드이펙트
4. 모든 기능의 유저플로우를 일괄 작성하고, 각 기능에 번호를 부여한다. (이 번호가 이후 유스케이스 번호 00N의 기준이 된다)
5. 최종 문서를 `/docs/userflow.md` 경로에 생성한다.

규칙:
- 발생할 수 있는 엣지케이스를 반드시 대응한다.
- 구체적인 UI 문구 등은 포함하지 않는다.
- 판단이 필요한 모호한 사항은 사용자에게 직접 묻지 말고, 최종 보고 마지막에 "Open Questions" 목록으로 정리해 반환한다. (사용자 질문은 메인 루프가 대행한다)
