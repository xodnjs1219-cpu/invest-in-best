# PRD: invest-in-best

밸류체인 마인드맵 기반 투자 인사이트 웹 서비스

> 본 문서는 `docs/requirement.md`의 확정 요구사항을 기반으로 작성되었으며, 요구사항에 명시된 범위를 벗어난 기능은 포함하지 않는다. 기술 스택은 이후 단계에서 확정되므로 본 문서는 특정 구현 기술을 확정하지 않는다.

---

## 1. 제품 개요

산업 또는 특정 기업을 중심으로 한 **밸류체인을 마인드맵(노드-엣지) 형태로 시각화**하고, 밸류체인 전체와 각 노드(시장 참여 주체)의 현황을 **대시보드**로 제공하는 웹 서비스.

클라이언트가 보는 모든 데이터는 **자체 DB**에서 제공하며, 외부 API(OpenDART, SEC EDGAR, 토스증권 Open API)는 **DB 적재용 배치로만** 사용한다. 국내(KRX)와 미국 상장기업을 동시에 지원하고, UI는 그래프 중심의 사용자 친화적 화면으로 구성한다.

### 핵심 목표

1. **밸류체인 시각화**: 산업/기업 중심 밸류체인을 노드-엣지 마인드맵으로 직관적으로 탐색하게 하며, 노드를 밸류체인 단계 등으로 그룹핑해 구조를 명확히 드러낸다.
2. **현황 대시보드**: 밸류체인 단위의 핵심 지표(가치총액, 구성 기업 매출 합계)와 각 시점(스냅샷) 기준 과거 추이를 제공한다.
3. **사용자 참여**: 로그인 사용자가 공식 체인을 복제하거나 빈 캔버스에서 자신만의 밸류체인을 구성·그룹핑·저장하게 한다.
4. **신뢰 가능한 데이터 운영**: 정기 배치 수집 + LLM 보조 관계 갱신(어드민 승인) + 스냅샷 이력으로 시점 조회 가능한 데이터를 유지한다.
5. **법적 고지 준수**: 데이터 출처·최종 수집 시각 표기, 이용약관·개인정보처리방침 게시, 투자 면책 문구 상시 노출.

### MVP 범위 요약

- 공식 밸류체인은 대표 산업 2~3개(예: 반도체, 2차전지)를 시드로 제공한다.
- 대시보드 지표는 **가치총액**과 **구성 기업 매출 합계** 2종으로 한정한다. (소비총액 제외)
- 소셜 로그인은 **Google 우선** 지원, 네이버·카카오는 확장 가능하도록만 설계한다.
- LLM 관계 제안은 **기존 노드 간 관계**로 한정한다. (신규 노드 제안은 차기 단계)
- 어드민 임명 UI는 제외하며, 최초 어드민은 환경변수 이메일로 시드한다.
- UI 언어는 **한국어 단일**로 제공한다. (영문 지원은 Non-Goals)

### 기준 정책 (전역)

- **기준 통화**: KRW. 환율(KRW↔USD)은 일 1회 수집·보존하며, 일별 지표는 해당 일자 환율, 분기 매출은 분기 말일 환율로 환산한다. UI에 기준 통화와 환산 기준을 표기한다.
- **시계열 최소 시작 시점**: 국내 재무 데이터는 OpenDART 제약상 **2015 사업연도 이후**만 제공한다.
- **시가총액 산출**: 직접 제공 API가 없어 `일별 종가 × 최신 상장주식수`로 자체 계산하고, 대시보드에 '주식수 기준일'을 주석으로 노출한다.
- **커버리지 표기**: 지표는 종목 마스터에 연결된 상장기업 노드만 합산하며, 대시보드에 "지표 반영 n/전체 m 노드" 커버리지를 표기한다.
- **규모 상한(상수 관리)**: 체인당 노드 최대 **100개**, 1인당 밸류체인 최대 **50개**. 상수로 관리해 조정 가능하게 한다.
- **면책**: 모든 데이터는 정보 제공 목적이며 투자 권유가 아님을 명시한다.

## 2. Stakeholders

- **비로그인 방문자 (Guest)**: 공식 밸류체인 열람, 기업 검색/상세 조회, 공식 체인 현황 대시보드 확인. 생성/편집/저장은 불가(로그인 유도).
- **로그인 사용자 (User)**: Guest 권한 + 밸류체인 생성/복제/편집/저장, 노드 그룹핑, 내 밸류체인 목록 관리, 기업 상세에서 내 소속 체인 확인, 계정 관리 및 회원 탈퇴.
- **어드민 (Admin)**: 공식 밸류체인 CRUD, LLM 관계 변경안 승인/거부, 배치 모니터링(조회), 관계 종류 마스터 관리. 어드민 라우트/API는 서버 측에서 role을 검증한다.
- **데이터/운영 파이프라인 (System)**: 시세·재무·공시·환율·장운영시간 정기 수집, 일별 체인 지표 사전 집계, LLM 공시 분석 및 검토 큐 적재, 최초 백필 수행. (사용자 직접 상호작용은 없으나 어드민 모니터링 대상)
- **서비스 운영/법무 (Internal)**: 이용약관·개인정보처리방침·투자 면책 문구 관리, 데이터 출처 표기 정책 관리. (MVP에서는 정적 페이지 형태로 반영)

**역할 정책**: 사용자 프로필에 `role`(user/admin)을 둔다. 최초 어드민은 환경변수의 어드민 이메일로 시드하며, 어드민 임명 UI는 MVP에서 제외한다. 공식 밸류체인 열람은 비로그인 허용, 생성/저장/편집은 로그인 필수다.

## 3. 포함 페이지

1. **메인/탐색 페이지**
   - 공식 밸류체인 카드 목록: 이름, 기준(산업/기업), 노드 수, 가치총액.
   - 로그인 시 내 밸류체인 목록 표시.
   - "새 밸류체인 만들기" 진입점 (비로그인 시 로그인 유도).
   - 기업 검색: **티커 또는 종목명(부분 일치)**으로 **통합 검색**. 결과 항목에 **시장 배지(KRX/US)**를 표시하고 **시장 필터**를 제공한다. 결과 선택 시 기업 상세로 이동.

2. **밸류체인 뷰 페이지**
   - 마인드맵 렌더링: 주체=노드, 관계=엣지(관계 종류 라벨 표시).
   - **그룹 표시**: 같은 그룹의 노드들을 시각적 묶음(배경 영역/클러스터)과 그룹 라벨로 표시한다.
   - 노드 클릭: 상장기업 노드 → 기업 상세 이동, 그 외 주체 → 정보 패널 표시.
   - 대시보드 패널: 가치총액/구성 기업 매출 합계의 현재값 + 과거 추이 그래프(시총 일단위, 매출 분기단위) + 커버리지 표기. 매출 합계는 KRW 환산값으로 표시하고 툴팁에 환산 기준(분기 말일 환율)을 명시한다.
   - **시점 타임라인**: 일 단위 슬라이더/달력 위에 **구조 변경 스냅샷 마커**를 함께 표시한다. 임의 날짜 선택 시, 그 날짜 이전 마지막 스냅샷 기준으로 구조(노드/관계/그룹)를 복원하고 지표는 해당 일자 값을 표시한다.
   - 폴백: 시세 데이터 미수집/장애 시에도 구조·재무 정보만으로 동작.
   - 매출 합계 툴팁: 단계 간 거래 중복·비관련 사업부 매출 포함 가능성 명시.

3. **밸류체인 생성/편집 페이지** (로그인 필수, 소유자만)
   - 빈 캔버스 신규 생성 또는 공식 체인 복제 후 편집(복제본은 복제 시점의 완전 독립 사본, 출처는 메타데이터로만 기록, **복제 후 편집 제약 없음**).
   - 밸류체인 기준 선택: 산업 중심 / 특정 기업 중심.
   - 노드 추가: 종목 검색(티커/종목명)으로 상장기업 연결 또는 자유 주체(소비자/정부 등) 생성.
   - 관계 설정: 두 노드 선택 후 활성화된 관계 종류 중 선택.
   - **노드 그루핑**: 노드 다중 선택 후 그룹 생성(이름 지정)/그룹 해제/그룹 이름 변경, 노드의 그룹 이동. 한 노드는 최대 1개 그룹에 속한다(중첩 없음).
   - 규모 상한 안내: 체인당 노드 100개, 1인당 체인 50개 도달 시 안내.
   - 저장 시 본인 소유 비공개 밸류체인으로 저장.

4. **기업 상세 페이지**
   - 기업 정형 정보: 회사명, 티커, 시장, 업종, 대표자, 설립일, 홈페이지 등 API 제공 필드. (서술형 개요 미제공)
   - 재무제표: 분기별 주요 계정(매출/영업이익/순이익 등) 표 + 그래프.
   - 주요 공시 목록(최신순, 원문 링크).
   - 주가 차트(일봉) + 시가총액 추이.
   - 소속 밸류체인 목록: 이 기업이 포함된 체인과 각 현황 요약. 비로그인 시 공식 체인만, 로그인 시 내 체인 포함.
   - 데이터 출처 및 최종 수집 시각 표기.

5. **로그인/회원가입 · 계정 페이지**
   - 이메일 회원가입/로그인 + Google 소셜 로그인(네이버·카카오 확장 대비).
   - 비밀번호 재설정: 표준 이메일 재설정 링크 플로우(세부 정책은 구현 단계에서 정의).
   - **계정 메뉴**: 회원 탈퇴 진입(탈퇴 시 사용자 생성 밸류체인 즉시 삭제).

6. **어드민 페이지** (role=admin, 서버 측 검증)
   - 공식 밸류체인 CRUD (시드: 반도체·2차전지 등 2~3개 산업). 노드·관계·그룹 구성 편집 포함.
   - LLM 관계 변경안 검토 큐: 근거 공시와 함께 제안 표시, 승인/거부.
   - 배치 작업 모니터링: 수집 작업별 최근 실행 상태/실패 로그 **조회**(MVP는 조회 전용, 수동 재실행은 차기 단계).
   - 관계 종류 마스터 관리: 추가/이름 변경/비활성화(물리 삭제 금지).

7. **약관/정책 페이지**
   - 이용약관, 개인정보처리방침, 투자 면책 문구 페이지. (데이터 보관·파기 세부 수치는 개인정보처리방침 문서에서 규정)
   - 전 페이지 푸터에 면책 요약 + 각 정책 링크 상시 노출.

## 4. 사용자 여정 (User Journey)

각 여정은 **타겟 유저 segment**와 이동하는 **페이지**를 명시한다.

### 4.1 비로그인 탐색자 (Guest — 산업/종목을 훑어보는 개인 투자자)

- **Segment**: 특정 산업의 구조나 관련 종목을 빠르게 파악하려는 리테일 투자자, 가입 전 서비스 가치를 확인하려는 방문자.
- **여정**:
  1. **메인/탐색 페이지** 진입 → 공식 밸류체인 카드 목록 확인.
  2. 관심 산업 카드 선택 → **밸류체인 뷰 페이지**에서 마인드맵/그룹 클러스터/대시보드(가치총액·매출 합계·커버리지) 확인.
  3. **밸류체인 뷰 페이지**의 시점 타임라인(슬라이더 + 스냅샷 마커)으로 과거 시점의 구조·그룹·지표 비교.
  4. 관심 상장기업 노드 클릭 → **기업 상세 페이지**에서 재무/공시/주가/소속 공식 체인 확인.
  5. "새 밸류체인 만들기" 시도 → 로그인 유도 → **로그인/회원가입 · 계정 페이지**로 전환.

### 4.2 로그인 파워 유저 (User — 자신만의 밸류체인을 구성하는 사용자)

- **Segment**: 공식 체인을 참고해 나만의 투자 논리로 밸류체인을 만들고 그룹핑·관리하려는 사용자.
- **여정**:
  1. **로그인/회원가입 · 계정 페이지**에서 이메일 또는 Google로 가입/로그인.
  2. **메인/탐색 페이지**에서 내 밸류체인 목록 확인 및 "새 밸류체인 만들기" 진입.
  3. **밸류체인 생성/편집 페이지**에서 빈 캔버스 신규 생성 또는 공식 체인 복제(제약 없는 자유 편집) 후 기준(산업/기업) 선택.
  4. 노드 추가(종목 검색으로 상장기업 연결 또는 자유 주체 생성) → 두 노드 간 활성 관계 종류 선택.
  5. 노드 다중 선택 → **그룹 생성/이름 지정/그룹 이동**으로 단계별 클러스터 구성 → 저장(본인 소유 비공개).
  6. **밸류체인 뷰 페이지**에서 내 체인의 그룹 클러스터·대시보드·타임라인 확인.
  7. 필요 시 **기업 상세 페이지**에서 내 소속 체인 포함 현황 확인, 또는 **계정 페이지(계정 메뉴)**에서 회원 탈퇴(내 체인 즉시 삭제).

### 4.3 기업 리서치 유저 (Guest/User — 개별 종목 리서치가 목적)

- **Segment**: 특정 종목의 재무·공시·주가와 그 기업이 속한 밸류체인 맥락을 함께 보려는 사용자.
- **여정**:
  1. **메인/탐색 페이지** 검색창에 티커/종목명(부분 일치) 입력 → 시장 배지(KRX/US)·시장 필터로 결과 선별.
  2. 검색 결과 선택 → **기업 상세 페이지**에서 정형 정보/분기 재무/공시/일봉·시총 추이 확인.
  3. 소속 밸류체인 목록에서 체인 선택 → **밸류체인 뷰 페이지**로 이동해 산업 맥락 확인.
  4. (로그인 시) **밸류체인 생성/편집 페이지**에서 해당 기업을 포함한 나만의 체인 구성.

### 4.4 어드민 (Admin — 공식 체인·데이터 운영자)

- **Segment**: 서비스 데이터 품질과 공식 콘텐츠를 관리하는 운영자.
- **여정**:
  1. **로그인/회원가입 · 계정 페이지**에서 어드민 계정 로그인(서버 측 role 검증).
  2. **어드민 페이지 > 공식 밸류체인 CRUD**에서 시드 산업 체인 생성/수정(노드·관계·그룹 구성 편집, 모든 변경은 구조 변경 이벤트로 기록).
  3. **어드민 페이지 > LLM 관계 변경안 검토 큐**에서 신규 공시 기반 제안을 근거 공시와 함께 검토 → 승인/거부(승인 시 공식 체인에 반영).
  4. **어드민 페이지 > 관계 종류 마스터 관리**에서 관계 종류 추가/이름 변경/비활성화.
  5. **어드민 페이지 > 배치 모니터링**에서 수집 작업 상태/실패 로그 조회.

## 6. 데이터·지표 정책 (요약)

PRD 범위 내 화면·여정을 뒷받침하는 데이터 정책 요약. 상세 스키마/집계 로직은 후속 DB·유스케이스 단계에서 정의한다.

### 밸류체인·노드·관계·그룹

- **노드(주체)**: 기업(상장/비상장)/소비자/정부 등 시장 참여 주체. 상장기업 노드는 종목 마스터와 연결된다.
- **밸류체인 기준**: 산업 중심 또는 특정 기업 중심.
- **관계 종류**: 사전 정의 마스터(공급, 고객, 경쟁, 협력, 지분투자, 규제 등)에서 선택. 물리 삭제 금지, 비활성화(is_active) 처리(기존 엣지·과거 스냅샷 유지, 신규 선택만 차단).
- **노드 그룹**: 체인 내에서 노드들을 그룹으로 묶는다(예: 소재, 셀 제조, 완성차 등 밸류체인 단계 또는 임의 분류). 그룹은 이름을 가지며 해당 체인에 소속되고, 한 노드는 **최대 1개 그룹**에 속한다(중첩 없음). 뷰에서는 시각적 묶음+라벨로 표시하고, 편집에서는 생성/해제/이름 변경/노드 이동이 가능하다.
- **공식 체인**: 전체 공개, 어드민 관리, 시드 산업 2~3개.
- **사용자 체인**: 본인만 열람. 공식 체인 복제본은 복제 시점의 완전 독립 사본(동기화·알림 없음, 편집 제약 없음).

### 시계열·이력

- 재무제표: 분기 단위. 주가/시가총액: 일 단위(시간별 수집 후 일별 집계 보존, 시간별 원본 30일 보존).
- 밸류체인 구조: 변경 이벤트 시점마다 스냅샷 기록 → 특정 과거 시점의 노드/관계/**그룹 구성**을 조회 가능. 시점 복원 시 그룹도 함께 복원한다.
- 구조 변경 유효 시점 = 어드민 승인/편집 시각, 근거 공시일은 메타데이터로 보관.
- 과거 지표는 각 시점 스냅샷 구성 기준으로 계산. 일별 체인 지표는 배치에서 사전 집계·저장하며, 구조 변경 시 과거 집계는 재계산하지 않고 변경 시점 이후부터 새 구성으로 집계.
- 타임라인 시점 조회 규칙: 임의 날짜 선택 시 그 날짜 이전 마지막 스냅샷으로 구조를 복원하고 지표는 해당 일자 값을 사용한다.

### 지표·통화

- **가치총액** = 구성 상장기업 시가총액 합산. **구성 기업 매출 합계** = 분기 매출 합산(시장규모 근사). 소비총액 MVP 제외.
- 합산 대상은 종목 마스터 연결 상장기업 노드만, 커버리지("반영 n/전체 m") 표기.
- 시가총액 = 일별 종가 × 최신 상장주식수(국내 OpenDART 주식총수현황, 미국 SEC dei 상장주식수). 상장주식수는 기준일과 함께 보관, 분기 갱신, '주식수 기준일' 주석 노출.
- 기준 통화 KRW, 환율 일 1회 수집. 일별 지표는 당일 환율, 분기 매출은 분기 말일 환율로 환산. UI는 KRW 환산값 표시 + 툴팁에 환산 기준 명시.

### 수집 범위·주기

- 정기 수집 대상: 종목 마스터 **전 종목**(편입 여부 무관, 시세·재무·공시 모두).
- 최초 적재 시 전 종목 과거 데이터 백필(일일 한도 내 분산 실행).
- 시세: 1시간 1회(시장별 개장 중에만) → 일별 집계, 종가는 장 마감 후 확정 배치 기록.
- 재무제표/공시/기업정보/환율/장운영시간: 1일 1회.
- 국내 분기 매출은 누적치 차감 산출(2Q=반기−1Q, 4Q=연간−3Q누적). 미국 분기 매출은 태그 폴백 체인(us-gaap 다중 태그 + ifrs-full) + Q4 차감으로 정규화, 태그 미매핑 기업은 집계 제외하되 제외 기업 수 표기.

### LLM 보조 업데이트

- 신규 공시를 LLM이 분석 → 관계 추가/변경/삭제안 제안 → 어드민 승인 시 반영.
- 대상: 공식 밸류체인 전용(사용자 체인은 수동 편집만). 제안 범위: 기존 노드 간 관계로 한정.
- 승인·수동 편집 포함 모든 공식 체인 변경은 구조 변경 이벤트로 기록.

### 연동 (배치 전용)

- **OpenDART** (국내 재무/공시/기업정보/주식총수): `docs/external/opendart.md`. 다중회사 조회·전체 매핑 API 우선 활용, 일일 한도 초과 시 분할·이월.
- **SEC EDGAR** (미국 재무/공시/상장주식수): `docs/external/sec-edgar-api.md`. User-Agent 헤더 필수.
- **토스증권 Open API** (시세/종목/환율/장운영시간, 국내+미국): `docs/external/tossinvest-openapi.md`. 시세 재배포 약관 확인 필수.
- **LLM** (공시 관계 추출): 공급자는 기술스택 단계에서 결정.
- 데이터 출처 표기: 금융감독원 DART, SEC EDGAR, 토스증권. 화면에 최종 수집 시각 표시.

## 7. 범위 제외 (Non-Goals / MVP 제외)

- **소비총액** 지표.
- **네이버·카카오 소셜 로그인** 실제 구현(확장 가능한 설계까지만).
- **LLM 신규 노드 제안**(제안은 기존 노드 간 관계로 한정, 신규 노드는 2단계).
- **어드민 임명 UI**(환경변수 이메일 시드로 대체).
- **어드민 배치 수동 재실행(retry) UI**(MVP는 조회 전용, 재실행은 2단계).
- **사용자 체인에 대한 LLM 보조 업데이트**(수동 편집만).
- **공식 체인 복제본의 원본 동기화/변경 알림**(복제는 독립 사본).
- **서술형 기업 개요 텍스트 제공**(API 제공 정형 필드만).
- **다국어(영문) UI**(MVP는 한국어 단일).
- **중첩 그룹(그룹 안의 그룹)**(한 노드는 최대 1개 그룹).

## 8. 가정 및 제약

- 클라이언트에 노출되는 모든 데이터는 자체 DB에서 제공되며, 외부 API는 배치 적재 용도로만 호출된다.
- 국내 재무 시계열은 2015 사업연도 이후만 제공 가능(OpenDART 제약).
- 시세는 시장 개장 중에만 수집되며, 시세 장애 시에도 구조·재무 정보 기반 폴백 화면이 동작해야 한다.
- 밸류체인 규모 상한은 체인당 노드 100개, 1인당 체인 50개이며 상수로 관리한다.
- 회원 탈퇴 시 계정과 사용자 생성 체인을 즉시 삭제한다. 데이터 보관·파기 세부 수치는 개인정보처리방침 문서에서 규정한다.
- 토스증권 Open API 키는 발급 예정이며, 시세 재배포 약관 확인이 구현 선행 과제다.
- 정기 수집은 각 API 일일 한도를 고려해 분할·이월 설계가 필요하다.
- 기술 스택(프론트/백엔드/DB/인증/LLM 공급자)은 본 PRD 이후 단계에서 확정된다.
