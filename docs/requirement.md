# Requirements: invest-in-best (밸류체인 기반 투자 인사이트 서비스)

## 개요

산업 또는 특정 기업 중심의 밸류체인을 마인드맵(노드-엣지) 형태로 시각화하고, 밸류체인 전체와 각 노드(시장 참여 주체)의 현황을 대시보드로 제공하는 웹 서비스.

- 클라이언트가 보는 모든 데이터는 자체 DB에서 제공하며, 외부 API(OpenDART, SEC EDGAR, 토스증권 Open API)는 DB 적재용 배치로만 사용한다.
- 대상 시장: 국내(KRX) + 미국 상장기업 동시 지원.
- UI는 그래프 중심의 사용자 친화적 화면으로 구성한다.
- 모든 데이터는 정보 제공 목적이며 투자 권유가 아님을 명시한다(면책).

## 페이지 구성

- **메인/탐색 페이지**: 공식 밸류체인 목록, 내 밸류체인 목록(로그인 시), 기업 검색창, "새 밸류체인 만들기" 진입점
- **밸류체인 뷰 페이지**: 마인드맵 시각화 + 현황 대시보드 + 시점 타임라인
- **밸류체인 편집 페이지**: 노드/관계 편집 캔버스 (로그인 필수)
- **기업 상세 페이지**: 기업 정형 정보/재무제표/공시/주가 + 소속 밸류체인 현황
- **로그인/회원가입 페이지**: 이메일 + 소셜 로그인
- **어드민 페이지**: 공식 체인 관리, LLM 관계 변경안 승인, 배치 모니터링, 관계 종류 마스터 관리
- **약관/정책 페이지**: 이용약관, 개인정보처리방침, 투자 면책 문구 (전 페이지 푸터 링크)

## 데이터

### 인증·권한
- 이메일 + 소셜 로그인. 소셜은 Google을 먼저 지원하고 네이버·카카오를 추후 추가할 수 있도록 확장 가능하게 설계한다. (구현 기술은 기술스택 단계에서 결정)
- 공식 밸류체인 열람은 비로그인 허용. 밸류체인 생성/저장/편집은 로그인 필수.
- 사용자 프로필에 role(user/admin)을 두고, 최초 어드민은 환경변수의 어드민 이메일로 시드한다. 어드민 라우트/API는 서버 측에서 role을 검증한다. 어드민 임명 UI는 MVP 제외.
- 회원 탈퇴 지원: 탈퇴 시 사용자 생성 밸류체인은 즉시 삭제한다.

### 밸류체인·노드·관계
- 노드(주체): 기업(상장/비상장)/소비자/정부 등 시장 참여 주체 무엇이든 가능. 상장기업 노드는 종목 마스터와 연결된다.
- 밸류체인 기준: 산업 중심 또는 특정 기업 중심.
- 관계 종류: 사전 정의된 마스터(공급, 고객, 경쟁, 협력, 지분투자, 규제 등)에서 선택. 마스터는 어드민이 관리하며 물리 삭제는 금지하고 비활성화(is_active)로 처리한다(기존 엣지·과거 스냅샷 유지, 신규 선택만 차단).
- 노드 그룹: 체인 내에서 노드들을 그룹으로 묶을 수 있다(예: 소재, 셀 제조, 완성차 등 밸류체인 단계 또는 임의 분류). 그룹은 이름을 가지며 해당 체인에 소속되고, 한 노드는 최대 1개 그룹에 속한다(중첩 그룹 없음).
- 공식(제공) 밸류체인: 전체 공개, 어드민이 관리. 시드로 대표 산업 2~3개(예: 반도체, 2차전지) 제공.
- 사용자 생성 밸류체인: 본인만 열람. 공식 체인 복제본은 복제 시점의 완전 독립 사본(출처는 메타데이터로만 기록, 동기화·알림 없음).

### 시계열·이력
- 재무제표: 분기 단위. 주가/시가총액: 일 단위(시간별 수집 후 일별 집계 보존, 시간별 원본은 30일 보존).
- 밸류체인 구조: 변경 이벤트 발생 시점마다 스냅샷 기록 — 특정 과거 시점에 어떤 주체가 속했고 어떤 관계였는지 조회 가능해야 한다. 스냅샷에는 노드/관계뿐 아니라 그룹 구성도 포함된다.
- 구조 변경의 유효 시점은 어드민 승인/편집 시각으로 하고, 근거 공시일은 메타데이터로 보관한다.
- 과거 시점 지표는 **각 시점의 스냅샷 구성 기준**으로 계산한다. 일별 체인 지표는 배치에서 사전 집계해 저장하고, 구조 변경 시 과거 집계는 재계산하지 않으며 변경 시점 이후부터 새 구성으로 집계한다.
- 국내 재무 데이터는 OpenDART 제약상 2015 사업연도 이후만 제공 — 시계열 UI의 최소 시작 시점으로 명시한다.

### 지표·통화
- 대시보드 지표(MVP): **가치총액**(구성 상장기업 시가총액 합산), **구성 기업 매출 합계**(분기 매출 합산, 시장규모 근사). 소비총액은 MVP 제외.
- 지표 합산 대상은 종목 마스터에 연결된 상장기업 노드만이며, 대시보드에 커버리지("지표 반영 n/전체 m 노드")를 표기한다.
- 매출 합계에는 단계 간 거래 중복·비관련 사업부 매출이 포함될 수 있음을 UI 툴팁으로 명시한다.
- 시가총액은 직접 제공 API가 없어 자체 계산한다: 일별 종가 × 최신 상장주식수(국내: OpenDART 주식총수현황, 미국: SEC dei 상장주식수). 상장주식수는 기준일과 함께 보관하고 분기 갱신하며, 대시보드에 '주식수 기준일'을 주석으로 노출한다.
- 기준 통화는 KRW. 환율(KRW↔USD)을 일 1회 수집해 일별 환율 시계열로 보존하고, 일별 지표는 해당 일자 환율, 분기 매출은 분기 말일 환율로 환산한다. UI에 기준 통화와 환산 기준을 표기한다.

### 수집 범위·주기
- 정기 수집 대상: 종목 마스터의 **전 종목** (밸류체인 편입 여부와 무관하게 시세·재무·공시 모두 정기 수집).
- 최초 적재 시 전 종목의 과거 데이터(일봉·분기 재무)를 백필하며, API 일일 한도 내에서 분산 실행한다.
- 시세: 1시간 1회. 시장별 장 운영시간(휴장일 포함)을 일 1회 수집해 개장 중에만 실행하고, 일별 집계는 해당 시장의 현지 거래일 기준, 종가는 장 마감 후 확정 배치로 기록한다.
- 재무제표/공시/기업정보/환율: 1일 1회.
- 국내 분기 매출은 누적치 차감 산출(2Q=반기−1Q, 4Q=연간−3Q누적). 미국 분기 매출은 매출 태그 폴백 체인(us-gaap 다중 태그 + ifrs-full)과 Q4 차감 로직으로 정규화하고, 태그 미매핑 기업은 집계에서 제외하되 제외 기업 수를 표기한다.

### LLM 보조 업데이트
- 신규 공시를 LLM이 분석해 관계 추가/변경/삭제안을 제안하고, 어드민 승인 시 반영한다.
- 적용 대상은 공식 밸류체인 전용(사용자 체인은 수동 편집만). 제안 범위는 기존 노드 간 관계로 한정(신규 노드 제안은 2단계).
- 승인·수동 편집을 포함한 모든 공식 체인 변경은 구조 변경 이벤트로 기록된다.

### 출처·법적 고지
- 데이터 출처 표기: 금융감독원 DART, SEC EDGAR, 토스증권. 화면에 최종 수집 시각을 표시한다.
- 개인정보처리방침 게시(회원가입 운영에 따른 법적 의무), 이용약관, 투자 면책 문구.

## 기능

### 메인/탐색 페이지
- 공식 밸류체인 카드 목록 표시 (이름, 기준[산업/기업], 노드 수, 가치총액).
- 로그인 시 내 밸류체인 목록 표시.
- "새 밸류체인 만들기" 진입점 제공 (비로그인 시 로그인 유도).
- 기업 검색: **티커 또는 종목명(부분 일치)**으로 검색, 결과 선택 시 기업 상세로 이동.

### 로그인/회원가입
- 이메일 회원가입/로그인 + Google 소셜 로그인 (네이버·카카오 확장 대비).
- 비밀번호 재설정 지원.
- 회원 탈퇴 (탈퇴 시 사용자 생성 체인 즉시 삭제).

### 밸류체인 뷰 페이지
- 마인드맵 렌더링: 주체=노드, 관계=엣지(관계 종류 라벨 표시).
- 그룹 표시 UI: 같은 그룹의 노드들을 시각적 묶음(배경 영역/클러스터)과 그룹 라벨로 표시한다.
- 노드 클릭: 상장기업 노드 → 기업 상세 페이지 이동, 그 외 주체 → 정보 패널 표시.
- 대시보드 패널: 가치총액/구성 기업 매출 합계의 현재값 + 과거 추이 그래프(시총 일단위, 매출 분기단위) + 커버리지 표기.
- 시점 타임라인: 과거 특정 시점 선택 시 그 시점의 구성 주체/관계(스냅샷)와 그 시점 기준 지표를 표시.
- 시세 데이터 미수집/장애 시에도 구조·재무 정보만으로 동작하는 폴백 표시.

### 밸류체인 생성/편집 페이지
- 빈 캔버스에서 신규 생성 또는 공식 체인 복제 후 편집.
- 밸류체인 기준 선택: 산업 중심 / 특정 기업 중심.
- 노드 추가: 종목 검색(티커/종목명)으로 상장기업 연결 또는 자유 주체(소비자/정부 등) 생성.
- 관계 설정: 두 노드 선택 후 활성화된 관계 종류 중 선택.
- 노드 그루핑: 노드 다중 선택 후 그룹 생성(이름 지정)/그룹 해제/그룹 이름 변경. 노드의 그룹 이동 가능.
- 저장 시 본인 소유 비공개 밸류체인으로 저장.

### 기업 상세 페이지
- 기업 정형 정보: 회사명, 티커, 시장, 업종, 대표자, 설립일, 홈페이지 등 API 제공 필드. (서술형 개요는 제공하지 않음)
- 재무제표: 분기별 주요 계정(매출/영업이익/순이익 등) 표 + 그래프.
- 주요 공시 목록(최신순, 원문 링크).
- 주가 차트(일봉) + 시가총액 추이.
- 소속 밸류체인 목록: 이 기업이 포함된 밸류체인과 각 체인의 현황 요약. 비로그인 시 공식 체인만, 로그인 시 내 체인 포함.
- 데이터 출처 및 최종 수집 시각 표기.

### 어드민 페이지
- 공식 밸류체인 CRUD (시드: 반도체·2차전지 등 2~3개 산업).
- LLM 관계 변경안 검토 큐: 근거 공시와 함께 제안 표시, 승인/거부.
- 배치 작업 모니터링: 수집 작업별 최근 실행 상태/실패 로그.
- 관계 종류 마스터 관리 (추가/이름 변경/비활성화 — 물리 삭제 금지).

### 배치 (데이터 수집·가공)
- 시세 수집: 1시간 1회, 시장별 개장 중에만 (토스증권 API, 국내+미국) → 일별 집계.
- 재무/공시/기업정보 수집: 1일 1회 (국내 OpenDART, 미국 SEC EDGAR).
- 환율·장 운영시간 수집: 1일 1회 (토스증권 API).
- 일별 체인 지표 사전 집계 (가치총액/매출 합계, 스냅샷 구성 기준).
- 신규 공시 → LLM 분석 → 관계 변경안 생성 → 어드민 검토 큐 적재.
- 최초 적재 시 전 종목 과거 데이터 백필 (일일 한도 내 분산 실행).
- 전 종목 정기 수집은 API 한도를 고려해 설계: OpenDART는 다중회사 조회(100사 묶음)·전체 매핑 API를 우선 활용하고, 일일 한도 초과 시 수집을 분할해 다음 실행으로 이월한다.

### 약관/정책
- 이용약관, 개인정보처리방침, 투자 면책 문구 페이지.
- 전 페이지 푸터에 면책 요약 + 각 정책 링크 상시 노출.

## 연동

- **OpenDART**: `docs/external/opendart.md` 참고. API 키 발급 완료(.env의 OPENDART_API_KEY). 일일 요청 한도는 마이페이지에서 확인 필요.
- **SEC EDGAR**: `docs/external/sec-edgar-api.md` 참고. 인증 불필요, User-Agent 헤더 필수.
- **토스증권 Open API**: `docs/external/tossinvest-openapi.md` 참고. 키 발급 예정 — 발급 시 약관의 시세 재배포 조항 확인 필수. 시세·종목·환율·장운영시간은 토큰만으로 호출 가능.
- **LLM(공시 관계 추출)**: 공급자는 기술스택 단계에서 결정.

### 선행 확인 과제 (구현 전)
1. 토스증권 WTS에서 Open API client_id/secret 발급 + 시세 재배포 약관 확인 → `.env` 추가
2. OpenDART 마이페이지에서 일일 요청 한도 확인
3. LLM API 키 준비 (공급자 확정 후)
