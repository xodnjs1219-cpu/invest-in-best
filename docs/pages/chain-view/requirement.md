# 밸류체인 뷰 페이지 (chain-view) — 요구사항 및 상태 정의

> 경로: `/valuechains/[chainId]` (+ `?at=YYYY-MM-DD` 시점 딥링크)
> 근거 문서: `docs/prd.md` 3장(밸류체인 뷰 페이지), `docs/userflow.md` 009~012, `docs/usecases/009~012/spec.md`, `docs/usecases/000_decisions.md`(C-1~C-8 — spec과 충돌 시 본 결정이 우선), `docs/database.md`, `docs/techstack.md`
> 본 문서는 페이지 요구사항(행동·데이터 흐름)과 **Level 1 상태 정의**(관리 상태 vs 파생 데이터, 상태 전환 테이블)를 다룬다. Flux/Context 설계는 `state_management.md` 참조.

---

## 1. 페이지 개요

밸류체인 하나를 **마인드맵(노드/엣지/그룹 클러스터) + 대시보드 패널(가치총액·매출 합계) + 시점 타임라인(슬라이더·달력 + 스냅샷 마커)** 으로 시각화하는 조회 전용 페이지다.

- **액터**: Guest(공식 체인), User(공식 체인 + 본인 소유 사용자 체인). 접근 불가(미존재·보관·비소유자)는 **404로 통일**되어 체인 존재 자체를 비노출한다(000_decisions C-2 — spec의 401/403 기술을 대체. FE는 방어적으로 401/403 응답도 동일한 "체인 없음" 폴백으로 처리한다).
- **사이드이펙트 없음**: 이 페이지의 어떤 상호작용도 서버 데이터를 생성/수정/삭제하지 않는다. 노드 위치 이동·그룹 접힘 등 캔버스 조작은 로컬 표시용이며 저장하지 않는다(저장은 편집 페이지 UC-015~018).
- **렌더링 기술**: React 19 + Next.js(App Router). 마인드맵은 `@xyflow/react`, 추이 차트는 `recharts`. 서버 상태는 TanStack Query, 클라이언트 상태는 Context + `useReducer`(techstack §2 기각 사유 참조 — Redux/Zustand 미사용).

### 페이지 구성 요소

| 영역 | 내용 |
|---|---|
| 마인드맵 캔버스 | 노드(상장기업/자유 주체), 엣지(관계 종류 라벨·방향 화살표), 그룹 클러스터(배경 영역 + 라벨, 빈 그룹은 라벨만 — C-1) |
| 대시보드 패널 | 가치총액 현재값 + 일 단위 추이, 구성 기업 매출 합계 현재값 + 분기 단위 추이, 커버리지("반영 n / 전체 m"), 제외 기업 수, 주석/툴팁(주식수 기준일 min~max — C-4, 환산 기준, 매출 중복 안내, 기준 통화 KRW) |
| 시점 타임라인 | 일 단위 슬라이더/달력(2015-01-01 ~ 오늘, Asia/Seoul — C-6) + 구조 변경 스냅샷 마커, 시점 조회 중 배지 + "최신으로 돌아가기" |
| 노드 정보 패널 | 자유 주체 노드 클릭 시: 이름·주체 유형·설명 메모(+그룹 부가 표시). 상장기업 종목 해석 실패 시: 폴백 패널 |
| 하단 표기 | 데이터 출처(금융감독원 DART, SEC EDGAR, 토스증권) + 최종 수집 시각(batch_runs 최근 성공 — C-3) |
| 편집 진입 | `chain.isOwner=true`인 사용자 체인일 때만 편집 버튼 노출(이동만, 편집은 별도 페이지) |

## 2. 사용자가 할 수 있는 행동과 데이터 흐름

### 행동 A. 페이지 진입 (최신 조회)

1. 사용자가 카드 목록/기업 상세/직접 링크로 진입한다.
2. FE가 병렬로 3계열 데이터를 로드한다 — 각 로드는 **독립 실패** 처리(구조 실패 ≠ 지표 실패).
   - 구조: `GET /api/valuechains/{chainId}` → 최신 스냅샷의 그룹/노드/엣지 + 체인 헤더 + 데이터 출처·수집 시각
   - 타임라인 메타: `GET /api/valuechains/{chainId}/timeline` → 선택 가능 범위 + 스냅샷 마커 목록
   - 지표: `GET /api/valuechains/{chainId}/metrics/daily`, `GET .../metrics/quarterly` (기본 기간 = 최근 1년 — C-5)
3. 마인드맵 렌더링: 저장 좌표(`position`)가 있으면 그대로, NULL이면 자동 레이아웃 폴백(UC-009 E11).
4. `?at=` 딥링크 진입이면 최초부터 시점 복원 흐름(행동 F)으로 시작한다.

### 행동 B. 캔버스 조작 (줌/팬/노드 드래그/그룹 접기)

- 줌/팬: React Flow 내부(비제어) 상태로 위임 — 앱 상태로 관리하지 않는다.
- 노드 드래그 종료: 해당 노드의 **로컬 좌표 오버라이드**만 갱신한다. 서버 호출 없음.
- 그룹 클러스터 접기/펼치기: 접힌 그룹의 소속 노드를 숨기고 클러스터 요약만 표시(대량 노드 성능 대응 — UC-009 E4).
- 시점 전환(행동 F) 시 로컬 오버라이드·접힘 상태는 초기화된다(스냅샷 스코프가 바뀌므로 무효).

### 행동 C. 노드 클릭

1. 클릭 시 노드 상세를 조회한다: `GET /api/valuechains/{chainId}/nodes/{nodeId}` (nodeId는 스냅샷 스코프 — 과거 시점 조회 중이면 자동으로 그 시점 정보).
2. 응답의 `nodeKind`/`securityResolved`로 분기한다.
   - `free_subject` → 정보 패널 표시(이름/주체 유형/설명 메모 + 그룹 부가 표시, 미소속이면 생략).
   - `listed_company` + `securityResolved=true` → `/companies/{ticker}`로 라우팅. 과거 시점 조회 중이었다면 "기업 상세는 최신 데이터 기준" 시점 컨텍스트 안내를 함께 전달. 상장폐지/정지 종목은 상태 배지 안내(UC-011 E4).
   - `listed_company` + `securityResolved=false` → 폴백 패널(상세 이동 불가 안내, UC-011 E1).
3. 빠른 연속 클릭은 **마지막 클릭 기준**으로만 반영한다(진행 중 요청 무시 — UC-011 E10).
4. 조회 실패 시 패널 영역에만 오류 폴백 + 재시도를 표시하고 라우팅은 하지 않는다(UC-011 E9).

### 행동 D. 노드 패널 닫기

- 패널을 닫으면 선택 노드가 해제되고 캔버스 하이라이트도 사라진다. 서버 호출 없음.

### 행동 E. 대시보드 기간 범위 변경

1. 기간 프리셋(1M/3M/6M/1Y/3Y/MAX) 또는 커스텀 구간을 선택한다.
2. FE가 일별/분기 지표 API를 새 기간으로 재조회한다(`from`/`to`, `fromYear·fromQuarter`/`toYear·toQuarter`).
3. 하한은 `TIMESERIES_MIN_START_DATE`(2015-01-01, 상수)로 보정하고 기간 선택 UI도 하한을 제한한다(UC-010 E8). 미래 `to`는 오늘로 보정.
4. 차트는 **거래일만** 표시(결측 이월값은 집계에 반영, x축 제외), `null`(미산출)과 `0`(집계 결과 0)을 구분 표기.

### 행동 F. 타임라인 시점 선택 (슬라이더/달력/마커 클릭)

1. FE가 선택 값을 1차 검증한다(미래·2015-01-01 이전 선택 차단).
2. 시점 복원 API를 호출한다: `GET /api/valuechains/{chainId}/snapshot-at?date=D` → D 이전(당일 23:59:59 경계 포함 — C-6) 마지막 스냅샷의 구조 + 해당 일자 일별 지표 + 해당 분기 지표.
3. 성공 시:
   - URL 쿼리를 `?at=D`로 갱신(딥링크 재현 보장).
   - 마인드맵을 스냅샷 좌표 기준으로 재렌더링, 대시보드 **현재값**을 선택 시점 기준으로 교체.
   - 추이 그래프는 **전체 추이를 유지하고 선택 시점을 하이라이트**한다(재조회로 범위를 좁히지 않음 — C-7).
   - 시점 조회 중 배지(선택 날짜·기준 스냅샷 시각) + "최신으로 돌아가기" 진입점 표시.
   - 이월(carry-forward)·커버리지 표기 유지. 분기 지표 미존재 시 "미제공" 표기(null, 0과 구분 — C-8).
4. 실패 시:
   - `SNAPSHOT_NOT_FOUND`(첫 스냅샷 이전 날짜): "이전 스냅샷 없음" 안내 + **직전 화면 상태 유지**(선택 시점을 마지막 성공 시점으로 되돌림).
   - 네트워크/서버 오류: 오류 안내 + 재시도, 기존 화면(직전 시점) 유지.
5. 로딩 중에는 직전 구조를 계속 표시한다(빈 화면 금지).

### 행동 G. 최신으로 돌아가기

- 선택 시점을 해제하고 URL에서 `?at=`을 제거, 최신 스냅샷 구조·최신 지표로 복귀한다(구조 데이터는 캐시 재사용 가능).

### 행동 H. 오류 재시도

- 구조 로드 실패(500 등): 전체 캔버스 영역 오류 폴백 + 재시도 버튼(UC-009 E8). 지표·타임라인과 독립.
- 지표 로드 실패: **지표 패널만** 오류 폴백 + 재시도. 구조·타임라인은 정상 동작(시세 폴백 원칙 — PRD 3장).
- 체인 404: 안내 후 메인 유도(재시도 없음).

## 3. 데이터베이스 사용

이 페이지는 **DB에 직접 접근하지 않는다.** 모든 데이터는 Hono API(features/valuechains/backend) 경유이며, 전 구간 **SELECT 전용**이다. 백엔드가 조회하는 테이블(유스케이스 009~012 종합):

| 테이블 | 용도 |
|---|---|
| `value_chains` | 체인 헤더·접근 제어(chain_type/owner_id/is_archived)·기준 종목 |
| `chain_snapshots` | 최신/시점 스냅샷 식별, 타임라인 마커(`effective_at` 목록) |
| `snapshot_groups` / `snapshot_nodes` / `snapshot_edges` | 구조(그룹/노드/엣지 + 좌표) 복원 |
| `securities` | 상장기업 노드 표시 정보(티커/종목명/시장/상장상태) |
| `relation_types` | 엣지 라벨(최신 이름)·방향성 |
| `chain_daily_metrics` | 일별 가치총액·커버리지·carry-forward 시계열/현재값 |
| `chain_quarterly_metrics` | 분기 매출 합계·제외 기업 수 시계열/현재값 |
| `shares_outstanding` | 주식수 기준일(min~max) 주석 메타 |
| `batch_runs` | 최종 수집 시각(잡별 최근 성공 — C-3) |

외부 API(OpenDART/SEC EDGAR/토스증권)는 이 페이지의 요청 경로에서 호출되지 않는다(배치 적재 전용 — PRD 8장).

## 4. 상태 정의 (Level 1)

### 4.1 설계 원칙

1. **서버 상태와 클라이언트 상태를 분리한다.** API 응답(구조/지표/타임라인 메타/노드 상세)은 전부 TanStack Query 캐시가 소유하며, **reducer 상태에 복사·보관하지 않는다.**
2. **파생 가능한 값은 상태로 두지 않는다.** 화면 표시값 대부분은 `서버 캐시 + 소수의 클라이언트 상태`에서 렌더 시점에 계산한다.
3. 클라이언트 상태는 아래 6개 필드가 전부다.

### 4.2 관리 상태 목록 (클라이언트 상태 — Context + useReducer)

| # | 상태 | 타입 | 초기값 | 설명 |
|---|---|---|---|---|
| S1 | `timeline.selectedDate` | `IsoDate \| null` | URL `?at=` 파싱값(형식/범위 무효 시 `null`) | 선택 시점. `null` = 최신. 구조/지표 쿼리 키와 URL의 원천 |
| S2 | `timeline.lastAppliedDate` | `IsoDate \| null` | `null` | 마지막으로 **복원에 성공한** 시점. 복원 실패 시 되돌림 목적지(직전 화면 유지 요구의 근거 상태) |
| S3 | `nodePanel.selectedNodeId` | `string \| null` | `null` | 클릭된 노드(스냅샷 스코프 ID). 노드 상세 쿼리 키. `null` = 패널 닫힘 |
| S4 | `dashboard.range` | `MetricsRange` (프리셋 또는 커스텀 구간) | `{ kind:'preset', preset:'1Y' }` (C-5) | 지표 조회 기간. 실제 from/to 날짜는 파생 계산 |
| S5 | `canvas.localNodePositions` | `Record<nodeId, {x,y}>` | `{}` | 노드 드래그의 로컬 좌표 오버라이드(표시용, 미저장) |
| S6 | `canvas.collapsedGroupIds` | `string[]` | `[]` | 접힌 그룹 클러스터 목록 |

### 4.3 서버 상태 목록 (TanStack Query — reducer 밖)

| 쿼리 | 엔드포인트 | 쿼리 키 입력 |
|---|---|---|
| 최신 구조 | `GET /api/valuechains/{chainId}` | chainId (S1이 `null`일 때만 활성) |
| 시점 복원 구조+지표 | `GET .../snapshot-at?date=D` | chainId + S1 (S1이 날짜일 때만 활성) |
| 타임라인 메타 | `GET .../timeline` | chainId |
| 일별 지표 | `GET .../metrics/daily` | chainId + S4 파생 범위 + S1(`at`) |
| 분기 지표 | `GET .../metrics/quarterly` | chainId + S4 파생 범위 + S1(`at`) |
| 노드 상세 | `GET .../nodes/{nodeId}` | chainId + S3 (S3가 있을 때만 활성) |

### 4.4 화면에 보이지만 상태가 아닌 것 (파생 데이터)

| 표시 항목 | 계산 원천 |
|---|---|
| 렌더링용 노드/엣지/그룹 배열 | 활성 구조 쿼리 데이터 + (서버 좌표 ?? 자동 레이아웃) + S5 오버라이드 + S6 접힘 필터 |
| 활성 구조(최신 vs 시점 복원) 선택 | S1 === null ? 최신 구조 쿼리 : 시점 복원 쿼리 |
| 과거 시점 조회 중 여부·시점 배지 텍스트 | S1 + 복원 쿼리의 `snapshot.effectiveAt` |
| "최신으로 돌아가기" 버튼 노출 | S1 !== null |
| URL `?at=` 쿼리 문자열 | S1 (상태→URL 단방향 동기화, 별도 저장 없음) |
| 추이 차트 선택 시점 하이라이트 (C-7) | S1 + 지표 시계열 |
| 대시보드 실제 조회 구간(from/to, 분기 축) | S4 + 오늘 날짜 + `TIMESERIES_MIN_START_DATE` 하한 보정 (순수 함수) |
| 지표 현재값·시계열·커버리지 문자열("반영 n / 전체 m") | 지표 쿼리 데이터 |
| **시세 폴백 표시** (오류 폴백/집계 준비 중/종가 미확정/이월 표기/분기 "미제공") | 지표 쿼리의 error·빈 시계열·`annotations.isClosingConfirmed`·`isCarriedForward`·`null` 값 — 전부 파생, 별도 상태 없음 |
| 타임라인 마커·선택 가능 범위 | 타임라인 메타 쿼리 데이터 |
| 노드 패널 모드(닫힘/로딩/자유주체/폴백/오류) | S3 + 노드 상세 쿼리 status·`nodeKind`·`securityResolved` |
| 상장기업 클릭 시 라우팅 목적지 | 노드 상세 쿼리 데이터(`ticker`) — 실행은 효과(effect), 상태 아님 |
| 데이터 출처·최종 수집 시각 | 구조 쿼리의 `dataFreshness` |
| 편집 버튼 노출 | 구조 쿼리의 `chain.isOwner` |
| 로딩/오류 폴백 화면 | 각 쿼리의 status/error (독립 영역별) |
| 캔버스 줌/팬 뷰포트 | React Flow 내부 상태(라이브러리 위임 — 앱 상태 아님) |
| 드래그 중 실시간 노드 좌표 | React Flow 내부 상태(드래그 **종료** 시점에만 S5로 반영) |

### 4.5 상태 전환 테이블

| 상태 | 변경 조건(트리거) | 변경 내용 | UI 반영 |
