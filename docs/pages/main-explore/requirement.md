## 1. 페이지 개요

- **경로**: `/` (서비스 루트)
- **액터**: Guest(비로그인) / User(로그인) — 공개 페이지, 누구나 접근 가능
- **성격**: **조회 전용** 페이지. 어떤 데이터도 생성/변경하지 않는다(사이드이펙트 없음).
- **역할**:
  1. 공식 밸류체인 카드 목록 노출 (이름·기준[산업/기업]·노드 수·가치총액 요약)
  2. 로그인 시 내 밸류체인 카드 목록 노출 (공식 체인과 동일 카드 스키마 — 결정 B-2)
  3. 기업 통합 검색 (티커/종목명 부분 일치, 시장 배지 KRX/US, 시장 필터)
  4. "새 밸류체인 만들기" 진입점 (비로그인 시 로그인 유도)
  5. 푸터 면책 요약·정책 링크 (전역 레이아웃 책임, UC-025 연계 — 본 페이지 상태 설계 범위 외)

## 2. 화면 구성 요소

| 영역 | 내용 | 노출 조건 |
|---|---|---|
| 검색창 + 시장 필터 | 검색어 입력(디바운스 300ms), 필터 탭(전체/KRX/US) | 항상 |
| 검색 결과 패널 | 종목명·티커·시장 배지 목록, 폐지/정지 종목은 상태 배지 표기(결정 B-5), 더보기 | 확정 검색어가 최소 길이(1자, 결정 B-4) 이상일 때 |
| 공식 밸류체인 섹션 | 체인 카드 목록(생성일순 — 결정 B-1) + 더보기 | 항상 |
| 내 밸류체인 섹션 | 체인 카드 목록(최근 수정순 — 결정 B-1) + 더보기 | 로그인 상태일 때만 |
| 새 밸류체인 만들기 버튼 | 생성 흐름(UC-013/014) 진입점 | 항상 (비로그인 클릭 시 로그인 유도) |

## 3. 사용자가 할 수 있는 행동과 데이터 흐름

### 3.1 페이지 진입

1. FE가 `GET /api/valuechains/official?page&limit` 호출 → 공식 체인 카드 목록 + 페이지네이션(`hasMore`) 수신 → 카드 렌더링.
2. 로그인 상태이면 `GET /api/valuechains/mine?page&limit` 추가 호출 → "내 밸류체인" 섹션 렌더링.
3. 두 목록의 오류/로딩/빈 상태는 **영역별로 독립 처리**한다 (공식 목록 오류가 내 목록 렌더링을 막지 않음, 역도 동일).
4. 카드의 가치총액(`latestMetric`)이 `null`이면 "값 미표시" 처리하여 **0과 명확히 구분**하고, `isCarriedForward=true`면 이월 표기를 덧붙인다.

### 3.2 기업 통합 검색

1. 사용자가 검색창에 입력 → 입력값은 즉시 화면에 반영되고, **디바운스 300ms**(상수) 타이머가 재시작된다.
2. 마지막 입력 후 300ms 경과 시 입력을 **정규화**(앞뒤 공백 제거, 전각→반각, 대소문자 무시 전제)하고 확정 검색어로 커밋한다.
3. 확정 검색어가 최소 길이(1자) 이상이면 `GET /api/securities/search?q&market&page` 호출 → 결과 패널 렌더링(정렬: 정확 > 접두 > 부분, 페이지당 20건 — 서버 책임).
4. 확정 검색어가 빈 문자열이면 결과 패널을 닫고 체인 목록 뷰로 복귀한다. 원본 입력은 있으나 정규화 결과가 최소 길이 미만(예: 공백만)이면 "검색 미실행" 안내를 표시한다.
5. 시장 필터(전체/KRX/US) 변경 시 동일 검색어로 재조회하며, **결과는 1페이지부터 다시** 표시한다.
6. 결과 항목 선택 → `/companies/[ticker]`로 라우팅(UC-020, 결정 B-6).
7. 결과 0건은 오류가 아닌 "빈 결과 안내"로 표시하고, 500 오류는 "오류 안내 + 재시도"로 구분 표시한다.
8. 레이트리밋은 FE 디바운스만 적용한다(결정 B-7).

### 3.3 페이지네이션 (더보기)

- 공식 체인/내 체인/검색 결과 각각 독립적인 "더보기" 버튼을 가진다.
- 페이지당 건수: 체인 목록 20건(`CHAIN_LIST_PAGE_SIZE`), 검색 20건(`SEARCH_PAGE_SIZE`) — **서로 분리된 독립 상수**(결정 B-3), 하드코딩 금지.
- 더보기 선택 시 다음 페이지(`page+1`)를 요청하고 기존 목록 뒤에 **이어 붙인다**. 응답의 `hasMore=false`이면 더보기 버튼을 숨긴다.
- 검색어 또는 시장 필터가 바뀌면 검색 결과의 누적 페이지는 초기화된다(1페이지부터).

### 3.4 체인 카드 선택

- 카드 클릭 → 밸류체인 뷰 `/valuechains/[chainId]`(UC-009)로 라우팅. 상태 변화 없음(내비게이션만).

### 3.5 새 밸류체인 만들기

- 로그인 상태: `/valuechains/new`(UC-013/014 흐름)로 라우팅.
- 비로그인: 로그인 페이지로 유도하되 **원래 목적지 복귀 정보**(returnTo)를 유지한다(UC-002/003 연계). 즉시 리다이렉트 방식이므로 본 페이지의 클라이언트 상태는 필요 없다.

### 3.6 오류 재시도

- 공식/내 체인/검색 각 영역의 500 오류 시 "재시도" 버튼 노출 → 해당 영역의 쿼리만 재요청.
- 내 체인 401(세션 만료) 시 게스트 뷰로 전환(내 체인 섹션 미노출)하되 공식 목록은 유지 + 로그인 유도.

## 4. API 및 데이터베이스

FE는 **DB에 직접 접근하지 않으며** 모든 데이터는 Hono API를 경유한다. 본 페이지가 의존하는 API와 그 배후 테이블(모두 SELECT 전용):

| API | 배후 테이블 | 용도 |
|---|---|---|
| `GET /api/valuechains/official` | `value_chains`, `chain_snapshots`, `snapshot_nodes`, `chain_daily_metrics`, `securities` (RPC `list_chain_cards`) | 공식 체인 카드 목록 (`is_archived=false`) |
| `GET /api/valuechains/mine` (인증 필수) | 동일 (owner 필터) | 내 체인 카드 목록 (`owner_id=현재 사용자`) |
| `GET /api/securities/search` | `securities` (트라이그램 GIN 인덱스) | 통합 검색 (ticker/name/english_name 부분 일치) |

- INSERT/UPDATE/DELETE 없음. 가치총액은 배치(UC-029)가 사전 집계한 `chain_daily_metrics` 최신 행만 읽는다. 런타임 외부 API 호출 없음.

## 5. 관리 상태 vs 파생(Derived) 데이터

> 원칙: **상태는 최소한으로.** 파생 가능한 값은 상태로 두지 않는다. TanStack Query가 관리하는 서버 캐시는 클라이언트 상태(reducer)에 중복 보관하지 않는다.

### 5.1 관리해야 할 클라이언트 상태 (useReducer 대상)

| # | 상태 | 타입 | 초기값 | 상태인 이유 |
|---|---|---|---|---|
| S1 | `searchInput` | `string` | `''` | 검색창의 원본 입력값(제어 컴포넌트). 사용자 입력 시점에만 알 수 있음 |
| S2 | `submittedQuery` | `string` | `''` | 디바운스 300ms + 정규화를 거쳐 **확정된** 검색어. 시간 의존적이라 `searchInput`에서 동기적으로 파생 불가 → 별도 상태. 검색 API 요청 파라미터의 유일한 원천 |
| S3 | `marketFilter` | `'ALL' \| 'KRX' \| 'US'` | `'ALL'` | 사용자의 필터 선택. 서버 응답으로부터 파생 불가 |

이 3개가 전부다. 아래는 상태로 두지 않기로 한 후보와 그 근거:

| 후보 | 상태로 두지 않는 이유 |
|---|---|
| 공식/내 체인 목록, 검색 결과, `hasMore`, 누적 페이지 | **서버 상태** — TanStack Query(`useInfiniteQuery`) 캐시가 단일 소스. reducer에 복사하면 이중 관리 |
| 로딩/오류 플래그 | 쿼리 훅의 `isPending`/`isError`/`isFetchingNextPage`에서 파생 |
| 검색 결과 패널 표시 여부 | `submittedQuery` 길이에서 파생 (아래 D1) |
| 로그인 여부 | 전역 인증 세션(Supabase Auth `onAuthStateChange`)에서 파생 — 페이지 소유 상태 아님 |
| 페이지 번호 | `useInfiniteQuery`의 `pageParam`이 관리. 검색어/필터 변경 시 쿼리 키가 바뀌므로 자동으로 1페이지부터 시작 |
| 로그인 유도(새 체인 만들기) | 즉시 리다이렉트(returnTo 쿼리) — 내비게이션 사이드이펙트일 뿐 상태 아님 |

### 5.2 화면에 보이지만 상태가 아닌 것 (파생 데이터)

| # | 파생 데이터 | 파생 원천 | 계산 |
|---|---|---|---|
| D1 | `isSearchActive` (검색 결과 패널 표시 여부) | S2 | `submittedQuery.length >= MIN_SEARCH_QUERY_LENGTH(1)` |
| D2 | "검색 미실행" 안내 표시 여부 | S1, S2 | 원본 입력은 있으나 정규화 결과가 최소 길이 미만 |
| D3 | 공식/내 체인 카드 목록, 검색 결과 목록(누적) | 쿼리 캐시 | `data.pages.flatMap(p => p.items)` |
| D4 | 더보기 버튼 노출 여부 (영역별) | 쿼리 캐시 | `hasNextPage` (응답 `hasMore` 기반 `getNextPageParam`) |
| D5 | 로딩 스피너/스켈레톤, 오류+재시도, 빈 상태 안내 | 쿼리 상태 | `isPending` / `isError` / `성공 && items.length===0` |
| D6 | 가치총액 표시 문자열 (미표시·이월·커버리지 "반영 n/전체 m") | 응답 `latestMetric` | `null`→값 미표시(0과 구분), `isCarriedForward`→이월 표기, KRW 포맷팅 |
| D7 | 시장 배지(KRX/US), 폐지/정지 상태 배지 | 응답 항목 필드 | `market` 및 종목 상태 필드 매핑(결정 B-5) |
| D8 | 내 밸류체인 섹션 노출 여부 | 인증 세션 | `isAuthenticated` |
| D9 | 카드 기준 표기 (산업 중심/기업 중심+기업명) | 응답 `focusType`, `focusCompanyName` | 필드 매핑 |

## 6. 상태 전환 테이블

### 6.1 클라이언트 상태 전환 (reducer 관리)

| 상태 | 변경 조건 (트리거) | 변경 내용 | UI 반영 |
|---|---|---|---|
| `searchInput` | 사용자가 검색창에 타이핑/삭제 | 입력값으로 갱신 | 검색창에 즉시 반영, 디바운스 타이머 재시작 |
| `searchInput` | 검색 지우기(X) 클릭 | `''`로 초기화 | 검색창 비움 |
| `submittedQuery` | 마지막 입력 후 300ms 경과(디바운스 만료) | 정규화된 입력값으로 갱신 | ① 최소 길이 이상 → 검색 쿼리 키 갱신 → 결과 패널 표시·재조회(1페이지부터) ② 빈 문자열 → 결과 패널 닫힘, 체인 목록 뷰 복귀 ③ 원본은 있으나 최소 길이 미만 → "검색 미실행" 안내 |
| `submittedQuery` | 검색 지우기(X) 클릭 | `''`로 초기화 | 결과 패널 즉시 닫힘(디바운스 대기 없음) |
| `marketFilter` | 필터 탭(전체/KRX/US) 선택 | 선택값으로 갱신 | 필터 탭 활성 표시 변경. 검색 활성 중이면 쿼리 키 갱신 → 결과 1페이지부터 재조회 |
| `marketFilter` | 검색 지우기(X) 클릭 | `'ALL'`로 초기화 | 다음 검색이 전체 시장 기준에서 시작 |

### 6.2 (참고) 서버 상태 ↔ UI 매핑 — TanStack Query 관리, reducer 범위 외

| 쿼리 | 전환 조건 | UI 반영 |
|---|---|---|
| 공식 체인 목록 | 페이지 진입(마운트) | 로딩 스켈레톤 → 카드 목록 / 빈 상태 안내(0건) / 오류+재시도(500) |
| 내 체인 목록 | 로그인 상태에서 마운트 (`enabled: isAuthenticated`) | 동일. 0건이면 빈 상태 + 생성 유도. 401이면 섹션 미노출(게스트 뷰) + 로그인 유도 |
| 검색 결과 | `submittedQuery`/`marketFilter` 변경으로 쿼리 키 갱신 (`enabled: isSearchActive`) | 로딩 → 결과 목록 / 빈 결과 안내 / 오류+재시도. 400은 입력 오류 안내 |
| 각 목록 다음 페이지 | 더보기 클릭 → `fetchNextPage()` | `isFetchingNextPage` 동안 버튼 로딩 표시 → 목록 뒤에 이어 붙임 → `hasNextPage=false`면 버튼 숨김 |

## 8. 엣지케이스 요약 (상태 관점)

| 상황 | 상태/파생 처리 |
|---|---|
| 공식 체인 0건(시드 미적재) | 서버 상태 파생(D5) — 빈 상태 안내. 클라이언트 상태 변화 없음 |
| 내 체인 0건 | 빈 상태 + 생성 유도 (D5) |
| `latestMetric=null` / 이월 집계 | 표시 파생(D6) — 값 미표시로 0과 구분 / 이월 표기 |
| 검색어 공백만 입력 | S2 커밋 결과 빈 문자열 → 미실행 안내(D2), API 미호출 |
| 검색 중 필터 변경 | S3 갱신 → 쿼리 키 변경 → 누적 페이지 자동 초기화 |
| 세션 만료 중 내 체인 401 | 쿼리 오류 파생 → 게스트 뷰 전환, 공식 목록 유지 |
| DB 조회 실패(500) | 영역별 독립 오류 + 재시도 (D5) |
| 더보기 파라미터 오류(400) | 쿼리 오류 파생 → 오류 안내 |
