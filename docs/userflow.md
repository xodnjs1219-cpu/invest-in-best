# Userflow: invest-in-best

밸류체인 마인드맵 기반 투자 인사이트 웹 서비스의 기능 단위 유저플로우.

> 본 문서는 `docs/requirement.md`, `docs/prd.md`의 확정 범위를 기반으로 작성되었다. 각 유저플로우는 **입력(사용자 UI 입력·상호작용) / 처리(시스템 내부 로직) / 출력(사용자 피드백·사이드이펙트)** 3단계와 **엣지케이스**로 구성한다. 특정 구현 기술을 확정하지 않으며(기술스택 미정), 구체적 UI 문구는 포함하지 않는다.
> 각 기능에 부여된 번호는 이후 유스케이스 문서 번호(`docs/usecases/00N/`)의 기준이 된다.

---

## 용어·전제

- **액터**: `Guest`(비로그인 방문자), `User`(로그인 사용자), `Admin`(role=admin), `System`(배치 파이프라인, 사용자 직접 상호작용 없음).
- **공식 체인**: 어드민이 관리하는 전체 공개 밸류체인. **사용자 체인**: 본인만 열람하는 비공개 밸류체인.
- **상장기업 노드**: 종목 마스터에 연결된 노드(지표 합산 대상). **자유 주체 노드**: 소비자/정부/비상장 등 종목 마스터 미연결 노드.
- **구조 변경 이벤트**: 공식 체인의 노드/관계/그룹 구성 변경 시 스냅샷을 기록하는 이벤트. 유효 시점 = 어드민 승인/편집 시각.
- **상수(설정값)**: 체인당 노드 최대 100개, 1인당 체인 최대 50개, 시간별 시세 원본 보존 30일, 시계열 최소 시작 시점 2015 사업연도, 검색 결과 페이지당 20건, 비밀번호 재설정 토큰 만료 1시간·재요청 레이트 리밋(분당 1회/일 5회), 배치 실패 재시도 3회 지수 백오프, LLM 일일 분석 건수 상한 등은 상수로 관리한다(본 문서는 값 자체가 아니라 "상한 도달" 조건으로 기술).
- **기준 통화**: KRW. 일별 지표는 당일 환율, 분기 매출은 분기 말일 환율로 환산.
- **계정 열거 방지**: 회원가입 중복/로그인 실패/비밀번호 재설정 요청 등에서 계정 존재 여부를 노출하지 않는 통일 문구를 사용한다(존재 여부와 무관한 일관 응답).
- **결측 데이터 처리**: 시세/환율 결측 일자는 직전 관측값 이월(carry-forward)로 집계하고, 차트는 거래일만 표시한다.
- **스냅샷 원칙**: 공식·사용자 체인 모두 편집 저장 1회당 1스냅샷을 기록한다(LLM 승인은 승인 1건당 1스냅샷). 사용자 체인도 타임라인 복원을 지원한다.

# A. 인증·계정

## 001. 이메일 회원가입

- **개요**: Guest가 이메일/비밀번호로 신규 계정을 생성한다.
- **액터/권한**: Guest.

### 입력
- 이메일, 비밀번호(및 확인), 필수 약관(이용약관·개인정보처리방침) 동의 체크.
- 회원가입 제출 상호작용.

### 처리
1. 클라이언트에서 형식 검증(이메일 형식, 비밀번호 정책, 확인 일치, 필수 동의 여부).
2. 서버에서 이메일 중복 여부 및 정책 재검증.
3. 계정 생성: 비밀번호는 단방향 해시로 저장, `role`은 기본 `user`로 부여. 단, 가입 이메일이 환경변수의 어드민 시드 이메일과 일치하면 `admin`으로 부여.
4. 이메일 인증 **필수**: 인증 메일 발송 및 미인증 상태로 대기. 확인 메일의 인증 링크를 클릭해야만 서비스 이용이 가능하다.
5. 약관 동의 이력(버전/시각)을 기록.

### 출력
- 성공: 계정 생성 완료 피드백 + 인증 메일 발송 안내. 인증 완료 전까지는 미인증 상태로 로그인 시 인증 안내 화면으로 유도. 인증 후 로그인 상태 전환, 최초 진입 컨텍스트(예: "새 밸류체인 만들기"에서 넘어온 경우) 복귀.
- 사이드이펙트: 사용자 레코드 생성(미인증), 약관 동의 이력 저장, 인증 메일 발송.

### 엣지케이스
- 이미 가입된 이메일: 계정 열거 방지 통일 문구로 응답(존재 여부 비노출).
- 소셜(Google)로 이미 가입된 검증된 동일 이메일: 이메일 가입 시 자동 연동(병합) 처리.
- 비밀번호 정책 미충족/확인 불일치: 필드 단위 오류 표기, 제출 차단.
- 필수 약관 미동의: 제출 차단.
- 네트워크/서버 오류: 재시도 유도, 중복 생성 방지(동일 요청 idempotency).
- 봇/무차별 가입 시도: 레이트 리밋/캡차 등 남용 방지(정책 결정).

## 002. 이메일 로그인

- **개요**: Guest가 이메일/비밀번호로 인증한다.
- **액터/권한**: Guest.

### 입력
- 이메일, 비밀번호, 로그인 제출.

### 처리
1. 형식 검증 후 서버로 자격 증명 전송.
2. 서버가 계정 조회 및 해시 대조로 인증.
3. 이메일 미인증 계정이면 로그인 차단 후 인증 안내 화면으로 유도(인증 메일 재발송 진입 제공).
4. 성공 시 인증 세션/토큰 발급, 사용자 `role`·프로필 로드.
5. 로그인 전 접근하려던 보호 경로가 있으면 해당 경로 복귀 정보 처리.

### 출력
- 성공: 로그인 상태 전환, 원래 목적지 또는 메인으로 이동. 헤더가 로그인 상태 UI로 갱신.
- 실패: 자격 증명 불일치 안내(계정 열거 방지 통일 문구 준수).
- 사이드이펙트: 세션/토큰 저장, 최근 로그인 시각 갱신(정책).

### 엣지케이스
- 자격 증명 불일치: 실패 횟수 누적, 반복 실패 시 레이트 리밋/잠금(정책).
- 미인증(이메일 인증 대기) 계정: 인증 안내 화면으로 유도, 서비스 이용 차단.
- 소셜 전용으로 가입해 비밀번호가 없는 계정: 소셜 로그인 유도.
- 탈퇴 처리된 계정: 로그인 불가 안내.
- 토큰 만료/무효 상태에서의 재로그인: 정상 재발급.

## 003. Google 소셜 로그인

- **개요**: Guest가 Google 계정으로 로그인/가입한다. (네이버·카카오는 확장 대비 설계만, 실제 구현은 MVP 제외)
- **액터/권한**: Guest.

### 입력
- Google 로그인 버튼 클릭, 외부 동의 화면에서의 계정 선택·동의.

### 처리
1. OAuth 인가 요청 → 외부 동의 화면 이동.
2. 콜백으로 인가 코드 수신 → 서버에서 토큰 교환 및 프로필(이메일 등) 획득.
3. Google 프로필의 이메일 미제공/미검증이면 가입 차단 + 이메일 가입 안내 후 종료.
4. 기존 계정 매칭:
   - 소셜 식별자 기존 연결 있음 → 해당 계정 로그인.
   - 검증된 동일 이메일의 이메일 가입 계정 존재 → 자동 연동(병합) 후 로그인.
   - 신규 → 계정 생성(`role=user`, 어드민 시드 이메일이면 `admin`), 약관 동의 처리. (소셜 인증 계정은 이메일 인증 완료로 간주)
5. 세션/토큰 발급.

### 출력
- 성공: 로그인 상태 전환, 목적지/메인 이동.
- 사이드이펙트: (신규 시) 계정·약관 동의 이력 생성, 소셜 식별자 연결 저장.

### 엣지케이스
- 사용자가 외부 동의 화면에서 취소: 로그인 페이지로 안전 복귀.
- 인가 코드 만료/재사용/CSRF(state 불일치): 인증 거부, 재시도 유도.
- 이메일 미제공/미검증 프로필: 가입 차단 + 이메일 가입 안내.
- 외부 제공자 장애/타임아웃: 오류 안내, 이메일 로그인 대체 경로 제시.
- 검증된 동일 이메일 계정 존재: 자동 연동(병합) 처리(별도 충돌 안내 없음).

## 004. 비밀번호 재설정

- **개요**: 표준 이메일 재설정 링크 플로우로 비밀번호를 재설정한다.
- **액터/권한**: Guest.

### 입력
- (요청 단계) 이메일 입력·제출.
- (완료 단계) 재설정 링크 진입 후 새 비밀번호·확인 입력·제출.

### 처리
1. 요청 이메일에 대해 계정 존재 여부와 무관하게 동일한 응답(계정 열거 방지 통일 문구).
2. 계정 존재 시 만료 1시간의 일회성 재설정 토큰 생성 및 메일 발송. 재요청은 레이트 리밋(분당 1회/일 5회, 상수)으로 제한.
3. 링크 진입 시 토큰 유효성(존재/미사용/미만료) 검증.
4. 새 비밀번호 정책 검증 후 해시 갱신, 토큰 무효화, **재설정 성공 시 해당 사용자의 전체 세션 강제 로그아웃**.

### 출력
- 요청: 발송 안내(존재 여부와 무관한 일관 응답).
- 완료: 재설정 성공 피드백 → 로그인 유도.
- 사이드이펙트: 재설정 토큰 생성/소모, 비밀번호 변경, 전체 세션 강제 로그아웃.

### 엣지케이스
- 만료(1시간 초과)/이미 사용된/위조 토큰: 무효 안내, 재요청 유도.
- 소셜 전용 계정에 대한 재설정 요청: 계정 열거 방지 범위 내 일관 응답.
- 재설정 링크 요청 남용: 레이트 리밋(분당 1회/일 5회) 적용.
- 새 비밀번호가 정책 미충족/확인 불일치: 제출 차단.

## 005. 로그아웃

- **개요**: 로그인 사용자가 세션을 종료한다.
- **액터/권한**: User/Admin.

### 입력
- 로그아웃 상호작용.

### 처리
1. 현재 기기(세션)의 세션/토큰만 무효화(서버 세션 폐기 또는 클라이언트 토큰 제거 + 서버 블랙리스트 정책).
2. 인증 상태 초기화.

### 출력
- 비로그인 상태로 전환, 공개 페이지(메인 등)로 이동, 헤더 UI 갱신.
- 사이드이펙트: 현재 기기 세션/토큰 폐기(타 기기 세션은 유지).

### 엣지케이스
- 이미 만료/무효 세션에서 로그아웃: 멱등 처리(정상 종료 처리).
- 다중 탭/기기 로그인: **현재 기기(세션)만** 로그아웃, 타 기기 세션은 유지.

## 006. 회원 탈퇴

- **개요**: User가 계정을 탈퇴한다. 탈퇴 시 사용자 생성 밸류체인을 즉시 삭제한다.
- **액터/권한**: User(로그인 필수, 본인 계정).

### 입력
- 계정 메뉴에서 탈퇴 진입, 확인(2단계 확인/재인증 등) 상호작용.

### 처리
1. 로그인·본인 확인 및 재인증(정책).
2. **유일 Admin 여부 검증**: 탈퇴 요청자가 유일한 Admin이면 탈퇴를 차단하고 다른 Admin 지정 필요 안내(진행 중단).
3. 탈퇴 확정 시 트랜잭션으로 사용자 소유 밸류체인 및 종속 데이터(노드/관계/그룹/사용자 체인 스냅샷·집계) 즉시 삭제.
4. 계정 비활성/삭제 처리(개인정보처리방침의 보관·파기 규정에 따름). 탈퇴 후 동일 이메일 재가입은 즉시 허용.
5. 세션 무효화.

### 출력
- 탈퇴 완료 피드백 → 비로그인 상태로 메인 이동.
- 사이드이펙트: 사용자 체인 즉시 삭제, 계정 삭제/비활성, 세션 폐기.

### 엣지케이스
- 삭제 중 부분 실패: 트랜잭션 롤백, 재시도 유도(일관성 보장).
- 공식 체인·종목 마스터 등 공용 데이터는 삭제 대상 아님(사용자 소유 데이터만).
- 탈퇴 사용자가 유일한 Admin인 경우: 탈퇴 차단(다른 Admin 지정 전 탈퇴 불가 안내).
- 탈퇴 직후 동일 이메일 재가입: 즉시 허용.
- 진행 중 배치가 사용자 데이터 참조: 사용자 체인은 배치 대상 아님(전 종목 수집만)이므로 영향 없음.

# B. 탐색·검색

## 007. 메인/탐색 페이지 조회

- **개요**: 공식 밸류체인 목록·(로그인 시) 내 밸류체인 목록·검색 진입점·생성 진입점을 제공한다.
- **액터/권한**: Guest/User.

### 입력
- 메인 페이지 진입, 카드/버튼/검색창 상호작용.

### 처리
1. 공식 밸류체인 목록 조회(카드: 이름, 기준[산업/기업], 노드 수, 가치총액 요약).
2. 로그인 상태면 내 밸류체인 목록 추가 조회.
3. 각 카드의 요약 지표(가치총액)를 사전 집계값에서 로드.
4. "새 밸류체인 만들기" 진입점 노출(비로그인 시 클릭하면 로그인 유도).
5. 푸터에 면책 요약·정책 링크 노출(기능 025 연계).

### 출력
- 공식/내 체인 카드 목록 렌더링, 검색창·생성 진입점 표시.
- 카드 선택 시 밸류체인 뷰(009)로 이동, 생성 진입 시 013/014 흐름.

### 엣지케이스
- 공식 체인이 아직 없음(시드 미적재): 빈 상태 안내.
- 로그인 사용자의 내 체인 0개: 빈 상태 + 생성 유도.
- 가치총액 집계값 미존재/시세 장애: 커버리지/폴백 표기 또는 값 미표시(0과 구분).
- 비로그인 사용자의 "내 밸류체인" 영역: 미노출 또는 로그인 유도.
- 목록 대량(사용자 체인 최대 상한 근접): 페이지당 상한(상수) 기준 페이지네이션 + 더보기.

## 008. 기업 통합 검색

- **개요**: 티커 또는 종목명(부분 일치)으로 국내(KRX)·미국(US) 상장기업을 통합 검색한다.
- **액터/권한**: Guest/User.

### 입력
- 검색어 입력(티커/종목명), 시장 필터 선택(KRX/US/전체), 결과 항목 선택.

### 처리
1. 입력 정규화(공백/대소문자/전각 등) 및 최소 길이 검증.
2. 종목 마스터에서 티커·종목명 부분 일치 검색, 시장 필터 적용.
3. 결과에 시장 배지(KRX/US) 부여. 정렬은 **정확 일치 > 접두 일치 > 부분 일치** 순, 페이지당 20건(상수) + 더보기(페이지네이션).
4. 결과 항목 선택 시 해당 기업 상세로 라우팅.

### 출력
- 결과 목록(종목명, 티커, 시장 배지) 렌더링. 선택 시 기업 상세(020)로 이동.
- 사이드이펙트: 없음(조회 전용).

### 엣지케이스
- 검색어 공백/과단(최소 길이 미만): 검색 미실행 안내.
- 결과 없음: 빈 결과 안내.
- 동일 종목명이 시장 간 중복: 시장 배지로 구분.
- 자유 주체(비상장) 노드는 종목 마스터 미연결 → 검색 대상 아님(검색은 종목 마스터 기준).
- 대량 매칭: 페이지당 20건 표시 + 더보기/시장 필터 유도.
- 종목 마스터 미수집/장애: 오류 안내, 재시도.
- 부분 일치 남용(과도한 요청): 디바운스/레이트 리밋.

# C. 밸류체인 뷰

## 009. 밸류체인 뷰 조회 (마인드맵·그룹 클러스터 렌더링)

- **개요**: 밸류체인을 노드-엣지 마인드맵과 그룹 클러스터로 시각화한다.
- **액터/권한**: 공식 체인은 Guest/User, 사용자 체인은 소유 User만.

### 입력
- 특정 체인 뷰 진입(카드 선택/검색/직접 링크), 캔버스 조작(줌/팬/노드 위치 이동 등 표시 상호작용).

### 처리
1. 체인 접근 권한 검증(공식=전체 공개, 사용자 체인=소유자 여부).
2. 현재 시점 기준 구성(노드/관계/그룹) 로드. 시점 미지정 시 최신 상태.
3. 노드(주체), 엣지(관계 종류 라벨), 그룹(시각적 묶음+그룹 라벨)을 렌더링.
4. 데이터 출처·최종 수집 시각 표기 데이터 로드.
5. 대시보드 패널(010)·타임라인(012)과 동일 페이지에서 연계 로드.

### 출력
- 마인드맵(노드/엣지/관계 라벨) + 그룹 클러스터 렌더링, 출처·최종 수집 시각 표기.
- 사이드이펙트: 없음(조회 전용). 표시용 위치 조정은 저장하지 않음(편집은 016~018).

### 엣지케이스
- 존재하지 않는/삭제된 체인: 안내 후 메인 유도.
- 사용자 체인 비소유자 접근: 접근 거부.
- 노드/엣지 대량(상한 근접): 렌더링 성능 저하 → 가상화/클러스터 접힘 등 대응.
- 관계 종류가 비활성화(is_active=false)된 엣지: 기존 엣지·과거 스냅샷은 유지·표시.
- 고립 노드(연결 엣지 없음)/그룹 미소속 노드: 정상 표시.
- 그룹만 있고 소속 노드 0개: 표시 정책(빈 그룹 처리).
- 구조 데이터 로드 실패: 오류 폴백.

## 010. 밸류체인 대시보드 패널 조회

- **개요**: 가치총액·구성 기업 매출 합계의 현재값·과거 추이·커버리지를 표시한다.
- **액터/권한**: Guest/User(해당 체인 열람 권한 내).

### 입력
- 뷰 페이지 진입(009 연계), 지표 그래프 조작(기간 범위 등), 시점 선택(012 연계).

### 처리
1. 지표 합산 대상은 종목 마스터 연결 상장기업 노드로 한정.
2. 가치총액 = 구성 상장기업 시가총액 합산(일 단위, 시가총액 = 일별 종가 × 최신 상장주식수).
3. 구성 기업 매출 합계 = 분기 매출 합산(분기 단위). KRW 환산(일별=당일 환율, 분기=분기 말일 환율).
4. 커버리지 계산("지표 반영 n / 전체 m 노드").
5. 과거 추이: 사전 집계된 일별/분기 지표 시계열 로드(각 시점 스냅샷 구성 기준).
6. 주석/툴팁 데이터 구성: 주식수 기준일, 환산 기준, 매출 중복·비관련 사업부 포함 가능성, 기준 통화.

### 출력
- 현재값 + 과거 추이 그래프(시총 일단위/매출 분기단위), 커버리지 표기, 주석/툴팁(주식수 기준일·환산 기준·중복 매출 안내).
- 사이드이펙트: 없음.

### 엣지케이스
- 상장기업 노드 0개(전부 자유 주체): 커버리지 0/m, 지표 미산출 안내.
- 일부 노드만 지표 반영: 커버리지로 명시, 부분 합산 표시.
- 시세 데이터 미수집/장애: 시세 기반 지표 폴백(구조·재무 정보만으로 동작), 미확정 표기.
- 상장주식수 미갱신/기준일 과거: 주식수 기준일 주석으로 명시.
- 미국 매출 태그 미매핑 기업: 매출 합계에서 제외 + 제외 기업 수 표기(027 연계).
- 환율/시세 결측 일자: 직전 관측값 이월(carry-forward)로 집계된 값 표시, 차트는 거래일만 표시(029 연계).
- 과거 구조 변경 이전 구간: 해당 시점 스냅샷 구성 기준 값 사용(구조 변경분은 재계산하지 않음).
- 2015 이전 재무 구간: 최소 시작 시점 이전은 미제공.

## 011. 노드 클릭 상호작용

- **개요**: 노드 클릭 시 유형에 따라 기업 상세 이동 또는 정보 패널을 표시한다.
- **액터/권한**: Guest/User.

### 입력
- 노드 클릭.

### 처리
1. 노드 유형 판별(상장기업 노드 vs 자유 주체 노드).
2. 상장기업 노드 → 연결된 종목(티커) 기준 기업 상세로 라우팅.
3. 자유 주체 노드 → 정보 패널에 3개 필드 표시: 이름, 주체 유형(소비자/정부/비상장기업/기타), 설명 메모(선택). (그룹 소속은 부가 표시)

### 출력
- 상장기업: 기업 상세(020)로 이동. 자유 주체: 정보 패널 표시.
- 사이드이펙트: 없음.

### 엣지케이스
- 상장기업 노드지만 종목 매핑 유실/종목 마스터에서 제거: 상세 이동 실패 → 폴백 패널/안내.
- 자유 주체 노드 정보 패널: 이름·주체 유형·설명 메모(선택) 3개 필드 표시.
- 과거 시점(스냅샷) 조회 중 클릭: 자유 주체는 해당 시점 노드 정보 기준으로 패널 표시. 상장기업은 현재 기업 상세로 이동하되 조회 중이던 시점 컨텍스트를 안내(기업 상세 자체는 최신 데이터 기준).

## 012. 시점 타임라인 조회 (스냅샷 복원)

- **개요**: 일 단위 슬라이더/달력 + 스냅샷 마커로 과거 시점의 구조·지표를 복원한다. **공식 체인·사용자 체인 모두 지원**(공식은 승인/편집 이벤트 스냅샷, 사용자 체인은 저장마다 기록된 스냅샷 기준).
- **액터/권한**: Guest/User(해당 체인 열람 권한 내).

### 입력
- 타임라인 슬라이더/달력에서 임의 날짜 선택, 스냅샷 마커 선택.

### 처리
1. 선택 가능 범위 설정(최소 시작 시점 2015 사업연도 이후 ~ 최신). 스냅샷 마커 위치 계산·표시(공식=승인/편집 이벤트, 사용자 체인=저장 이벤트 기준).
2. 임의 날짜 선택 시 **그 날짜 이전 마지막 스냅샷** 기준으로 구조(노드/관계/그룹) 복원.
3. 지표는 **해당 일자 값**(사전 집계된 일별 지표) 로드. 분기 매출은 해당 시점이 속한 분기 값.
4. 마인드맵·그룹·대시보드를 복원된 시점 기준으로 재렌더링.

### 출력
- 선택 시점의 구조(스냅샷)·그룹·지표 표시, 마커로 구조 변경 지점 시각화.
- 사이드이펙트: 없음.

### 엣지케이스
- 첫 스냅샷 이전 날짜 선택: 이전 스냅샷 없음 → 최소 시점/미표시 안내.
- 미래/범위 밖 날짜: 선택 제한.
- 해당 일자 지표 집계값 결측(휴장/미수집): 직전 관측값 이월(carry-forward) 값 표시, 차트는 거래일만 표시.
- 사용자 체인: 저장 이벤트마다 기록된 스냅샷 범위 내에서 복원(공식 체인은 승인/편집 이벤트 기준). 두 유형 모두 타임라인 지원.
- 구조 변경(공식)/저장(사용자)이 없는 체인: 마커 없이 단일 구성 표시.
- 스냅샷과 집계 시점 경계(변경 당일): 유효 시점 규칙(어드민 승인/편집 시각)에 따른 경계 처리.

# D. 밸류체인 생성·편집

## 013. 밸류체인 신규 생성 (빈 캔버스)

- **개요**: User가 빈 캔버스에서 새 밸류체인을 시작하고 기준을 선택한다.
- **액터/권한**: User(로그인 필수).

### 입력
- "새 밸류체인 만들기" 진입, 기준 선택(산업 중심/기업 중심), 체인 이름 입력.

### 처리
1. 로그인 검증(비로그인 시 로그인 유도 후 복귀).
2. 1인당 체인 상한 도달 여부 사전 확인.
3. 빈 편집 캔버스 초기화(임시 작업 상태), 기준·이름 설정.
4. 이후 노드/관계/그룹 편집(015~017) 및 저장(018)으로 연결.

### 출력
- 편집 캔버스 진입(기준/이름 반영). 저장 전까지 미확정.
- 사이드이펙트: 저장 시점(018)에 실제 체인 생성. 편집 초안 자동 저장은 없음(MVP).

### 엣지케이스
- 비로그인 진입: 로그인 유도, 로그인 후 생성 흐름 복귀.
- 1인당 체인 상한 도달: 생성 차단 안내(기존 체인 삭제 유도).
- 이름 미입력/중복: 저장 시 검증. 동일 사용자 내 이름 중복 불허(전역/타 사용자 간은 허용).
- 저장 전 이탈: 자동 저장 없이 미저장 이탈 경고 다이얼로그 노출(MVP).

## 014. 공식 체인 복제

- **개요**: User가 공식 체인을 복제해 편집 가능한 독립 사본을 만든다.
- **액터/권한**: User(로그인 필수).

### 입력
- 공식 체인 뷰/목록에서 복제 상호작용.

### 처리
1. 로그인 검증, 1인당 체인 상한 확인.
2. 복제 시점의 공식 체인 최신 구성(노드/관계/그룹)을 완전 독립 사본으로 스냅샷 복사.
3. 출처(원본 체인 식별자·복제 시각)를 메타데이터로만 기록(동기화·알림 없음).
4. 소유자 = 현재 사용자, 비공개로 초기화. 이후 제약 없는 자유 편집 가능.

### 출력
- 복제된 사용자 체인의 편집 캔버스/뷰 진입.
- 사이드이펙트: 사용자 체인 신규 생성(독립 사본), 출처 메타데이터 저장.

### 엣지케이스
- 1인당 체인 상한 도달: 복제 차단.
- 복제본 이름이 동일 사용자 기존 체인과 중복: 자동 접미어 부여 또는 이름 변경 유도(동일 사용자 내 이름 중복 불허).
- 복제 중 원본 공식 체인이 변경/삭제됨: 복제는 복제 시점 스냅샷 기준으로 일관 생성.
- 노드 상한(원본이 상한 근접): 사본도 동일 상한 적용(초과 불가).
- 비로그인 복제 시도: 로그인 유도.
- 복제본은 원본과 동기화되지 않음(사양) — 이후 원본 변경 알림 없음.

## 015. 노드 추가/삭제

- **개요**: 편집 캔버스에서 상장기업 노드(종목 연결) 또는 자유 주체 노드를 추가/삭제한다.
- **액터/권한**: User(소유자), Admin은 공식 체인 편집(021)에서 동일 로직.

### 입력
- (추가) 노드 추가 상호작용 → 종목 검색(티커/종목명) 후 선택, 또는 자유 주체 유형·이름 입력.
- (삭제) 노드 선택 후 삭제 상호작용.

### 처리
1. 추가-상장기업: 종목 마스터 검색·선택 → 종목 연결 노드 생성. 동일 체인 내 동일 종목 노드는 1개만 허용(중복 추가 차단).
2. 추가-자유 주체: 주체 유형(소비자/정부/비상장기업/기타)·이름·설명 메모(선택)로 종목 미연결 노드 생성.
3. 노드 상한(체인당 100) 사전 검증.
4. 삭제: 대상 노드와 연결된 엣지 정리, 그룹 소속 해제.
5. 임시 편집 상태 갱신(확정은 저장 018).

### 출력
- 캔버스에 노드 추가/삭제 반영. 상한 근접 시 안내.
- 사이드이펙트: 저장 시 반영. 공식 체인의 경우 저장이 구조 변경 이벤트/스냅샷 생성(021 연계).

### 엣지케이스
- 노드 상한 도달: 추가 차단 안내.
- 동일 종목 노드 중복 추가: 한 체인에 동일 종목 노드는 1개만 허용 → 차단 안내.
- 종목 검색 결과 없음/매핑 실패: 추가 불가 안내.
- 연결 엣지가 있는 노드 삭제: 관련 엣지 자동 삭제 + 확인.
- 그룹 내 마지막 노드 삭제로 빈 그룹 발생: 빈 그룹 유지/자동 정리 정책.
- 자유 주체 필수 필드 미입력: 추가 차단.

## 016. 관계(엣지) 설정/편집/삭제

- **개요**: 두 노드 간 관계를 활성화된 관계 종류에서 선택해 설정/변경/삭제한다.
- **액터/권한**: User(소유자), Admin(공식 체인).

### 입력
- 두 노드 선택, 관계 종류 선택(활성 마스터 중), 관계 편집/삭제 상호작용.

### 처리
1. 출발/도착 노드 선택 검증. 자기 참조(동일 노드 간) 엣지는 불허. 동일 노드 쌍에 **같은 관계 종류 중복은 불허**, **서로 다른 관계 종류 병존은 허용**.
2. 활성화된(is_active=true) 관계 종류 목록에서 선택 강제(비활성 종류는 신규 선택 불가).
3. 엣지 생성/변경/삭제를 임시 편집 상태에 반영. 방향성 처리는 **관계 종류 마스터의 방향성 속성(유향/무향)** 정의를 따름.
4. 확정은 저장(018). 공식 체인은 구조 변경 이벤트로 기록(021).

### 출력
- 캔버스에 엣지(관계 종류 라벨) 반영.
- 사이드이펙트: 저장 시 반영/스냅샷.

### 엣지케이스
- 동일 노드 쌍에 같은 관계 종류 중복: 차단. 서로 다른 관계 종류 병존: 허용.
- 자기 참조(동일 노드 간) 엣지: 차단.
- 선택한 관계 종류가 편집 중 비활성화됨: 신규 선택 차단, 기존 엣지는 유지.
- 한쪽 노드가 삭제됨: 연결 엣지 자동 정리.
- 활성 관계 종류가 0개(마스터 전부 비활성): 관계 설정 불가 안내.

## 017. 노드 그루핑

- **개요**: 노드를 그룹으로 묶어 밸류체인 단계 등으로 클러스터링한다.
- **액터/권한**: User(소유자), Admin(공식 체인).

### 입력
- 노드 다중 선택, 그룹 생성(이름 지정)/그룹 해제/그룹 이름 변경/노드의 그룹 이동 상호작용.

### 처리
1. 다중 선택 노드로 그룹 생성(이름 필수), 그룹은 해당 체인 소속.
2. 한 노드는 최대 1개 그룹 소속(중첩 없음) — 다른 그룹으로 이동 시 기존 그룹에서 자동 제거.
3. 그룹 해제(그룹 삭제 시 소속 노드는 그룹 미소속으로 전환, 노드 자체는 유지).
4. 그룹 이름 변경.
5. 임시 편집 상태 반영, 저장 시 확정. 스냅샷에 그룹 구성 포함.

### 출력
- 캔버스에 그룹 클러스터/라벨 반영.
- 사이드이펙트: 저장 시 그룹 구성 저장(스냅샷 포함).

### 엣지케이스
- 이미 다른 그룹 소속 노드를 새 그룹에 포함: 기존 그룹에서 이동(중첩 금지 준수).
- 그룹 이름 미입력/중복: 저장 차단/중복 허용 정책.
- 빈 그룹(노드 0개) 생성/잔존: 유지 또는 자동 정리 정책.
- 그룹 해제 시 소속 노드 처리: 노드 유지, 그룹만 제거.
- 다른 체인 노드를 그룹에 포함 시도: 차단(그룹은 체인 내부 한정).

## 018. 밸류체인 저장

- **개요**: 편집 상태(노드/관계/그룹/기준/이름)를 본인 소유 비공개 체인으로 저장한다.
- **액터/권한**: User(소유자), Admin(공식 체인 저장은 021로 이관).

### 입력
- 저장 상호작용.

### 처리
1. 소유자·로그인 검증.
2. 무결성 검증: 노드 상한(100)·1인당 체인 상한(50)·이름 필수·**동일 사용자 내 이름 중복 불허**·엣지 참조 유효성·그룹 소속 일관성.
3. 신규 저장이면 체인 생성, 기존이면 갱신. 사용자 체인은 비공개.
4. **사용자 체인도 저장 1회당 1스냅샷**(노드/관계/그룹 구성)을 기록해 타임라인 복원을 지원(공식 체인은 021에서 구조 변경 이벤트로 기록).

### 출력
- 저장 완료 피드백 → 뷰 페이지 이동.
- 사이드이펙트: 체인 영속화, (해당 시) 스냅샷/이력 기록.

### 엣지케이스
- 상한 초과(노드/체인 수): 저장 차단 안내.
- 이름 중복(동일 사용자 내): 저장 차단 안내.
- 저장 충돌(동시 편집/스토리지 오류): 재시도, 낙관적 잠금/버전 검증(정책).
- 유효하지 않은 엣지·그룹 참조: 저장 차단, 오류 위치 표시.
- 네트워크 중단: 재시도 유도(자동 저장 없음 → 미저장 손실 경고).
- 비소유자/세션 만료: 저장 거부, 재로그인 유도.

- **액터/권한**: User(소유자).

2. 확인 시 트랜잭션으로 체인 및 종속(노드/관계/그룹/스냅샷/집계) 삭제.

### 출력

### 엣지케이스
- 비소유자/공식 체인 삭제 시도: 거부(공식 체인은 Admin만, 021).

# E. 기업 상세

## 020. 기업 상세 조회

- **개요**: 상장기업의 정형 정보/재무/공시/주가/소속 밸류체인을 표시한다.
- **액터/권한**: Guest/User(소속 체인 노출 범위는 권한에 따름).

### 입력
- 검색 결과 선택/노드 클릭/직접 링크(티커)로 진입, 재무·주가 그래프 조작.

### 처리
1. 티커로 기업 식별, 종목 마스터·정형 정보 로드(회사명/티커/시장/업종/대표자/설립일/홈페이지 등 API 제공 필드).
2. 분기별 재무(매출/영업이익/순이익 등) 표+그래프 로드(2015 사업연도 이후, 국내 누적 차감·미국 태그 정규화 반영값).
3. 주요 공시 목록(최신순, 원문 링크) 로드.
4. 주가 차트(일봉) + 시가총액 추이(일별 종가 × 최신 상장주식수) 로드.
5. 소속 밸류체인 목록: 비로그인=공식 체인만, 로그인=공식+내 체인 포함, 각 현황 요약.
6. 데이터 출처(DART/SEC/토스증권)·최종 수집 시각 표기.

### 출력
- 정형 정보/재무/공시/주가/시총 추이/소속 체인/출처·수집 시각 렌더링.
- 소속 체인 선택 시 밸류체인 뷰(009)로 이동.
- 사이드이펙트: 없음.

### 엣지케이스
- 존재하지 않는/미상장/상장폐지 티커: 안내(폐지 시 과거 데이터 표시 정책).
- 재무 데이터 결측(신규 상장/미수집/2015 이전): 부분 표시 + 결측 안내.
- 미국 매출 태그 미매핑: 매출 항목 제외/주석.
- 시세 미수집/장애: 주가·시총 폴백, 정형·재무만 표시.
- 상장주식수 미갱신: 시총 추이에 기준일 주석.
- 소속 체인 없음: 빈 목록 안내.
- 비로그인의 내 체인 노출 시도: 공식 체인만 노출.
- 서술형 개요는 미제공(사양) — 정형 필드만 표시.

# F. 어드민

## 021. 공식 밸류체인 관리 (CRUD)

- **개요**: Admin이 공식 밸류체인을 생성/수정/삭제하며, 노드·관계·그룹 구성을 편집한다. 모든 변경은 구조 변경 이벤트로 기록된다.
- **액터/권한**: Admin(서버 측 role 검증).

### 입력
- 어드민 공식 체인 CRUD 진입, 노드/관계/그룹 편집(015~017 로직 재사용), 저장·삭제 상호작용.

### 처리
1. 서버 측 role=admin 검증(라우트/API).
2. 생성/수정: 노드·관계·그룹 편집을 적용하고 저장.
3. 저장 시 **구조 변경 이벤트 기록** — **저장 1회당 1스냅샷**(노드/관계/그룹), 유효 시점 = 편집 시각, 근거 공시일은 메타데이터로 보관(해당 시).
4. 삭제: **물리 삭제 금지, 보관(비공개 전환)** 처리 — 기존 복제본·스냅샷에는 영향 없음.
5. 변경 시점 이후부터 새 구성으로 지표 집계(구조 변경분 과거 재계산 없음, 029 연계).

### 출력
- 공식 체인 반영, 뷰/목록 갱신, 구조 변경 이벤트/스냅샷 기록.
- 사이드이펙트: 스냅샷·이벤트 저장, 이후 배치 집계 대상 구성 변경.

### 엣지케이스
- 비-Admin 접근: 서버 측 거부(클라이언트 우회 방지).
- 노드 상한 초과: 차단.
- 비활성 관계 종류 신규 선택 시도: 차단(기존 엣지 유지).
- 동시 편집 충돌: 버전 검증/잠금.
- 시드 미존재 상태에서 최초 생성: 정상 생성.
- 스냅샷 생성 단위: 저장 1회당 1스냅샷(공식/사용자 동일).
- LLM 승인 반영(022)과 수동 편집 동시 발생: 승인/편집 시각 순으로 직렬 처리해 이벤트 정합성 유지.
- 공식 체인 삭제: 물리 삭제 대신 보관(비공개 전환)으로 처리.

## 022. LLM 관계 변경안 검토 (승인/거부)

- **개요**: Admin이 LLM이 생성한 공식 체인 관계 변경안을 근거 공시와 함께 검토해 승인/거부한다.

### 처리
1. role 검증 후 대기 상태 제안 큐 로드(제안: 기존 노드 간 관계 추가/변경/삭제 + 근거 공시).
2. 승인: 해당 변경을 공식 체인에 반영 → **승인 1건당 1스냅샷**의 구조 변경 이벤트 기록(021 로직), 제안 상태=승인. 동일 체인에 수동 편집과 동시 발생 시 승인 시각 순으로 직렬 처리.
3. 거부: 반영 없이 제안 상태=거부(사유 기록 정책).
4. 제안 대상 노드/관계가 현재 체인과 불일치(그새 삭제/변경)면 충돌 처리.

### 엣지케이스
- 제안이 참조하는 노드/관계가 이미 삭제됨: 적용 불가 → 자동 무효/재검토 표시.

- 비활성 관계 종류를 제안: 적용 차단/대체 선택.
- 중복 제안(동일 변경 다건): 병합/중복 표시.
- 동시 다중 어드민 처리: 처리 잠금.
- 대상이 사용자 체인: 배제(LLM 보조는 공식 체인 전용).

## 026. 시세 수집 배치

- **개요**: 종목 마스터 전 종목의 시세를 1시간 1회(시장별 개장 중에만) 수집하고 일별 집계한다.
- **액터/권한**: System.

### 입력(트리거·소스)
- 시간별 스케줄 트리거, 시장별 장 운영시간(028에서 수집한 개장 여부/휴장일), 토스증권 Open API 시세(국내+미국), 대상=전 종목.

### 처리
1. 실행 시각 기준 각 시장의 개장 여부 판정 → 개장 중 시장만 수집.
2. 전 종목 시세 조회(배치·페이지네이션, API 한도 준수).
3. 시간별 원본 저장(보존 30일). 일별 집계 갱신(현지 거래일 기준).
4. 장 마감 후 확정 배치에서 종가 확정 기록.
5. 실행 상태/실패/한도 이월을 모니터링에 기록.

### 출력(적재·사이드이펙트)
- 시간별 시세 원본(30일 보존), 일별 집계, 확정 종가 적재. 최종 수집 시각 갱신.

### 엣지케이스
- 휴장일/조기 마감/서머타임(미국): 장 운영시간 데이터 기준 스킵/조정.
- API 한도 초과: 분할·다음 실행 이월.
- 부분 실패(일부 종목 결측): 종목 단위 3회 지수 백오프 재시도, 최종 실패는 부분 성공 기록 후 다음 주기 재포함. 일별 집계는 직전 관측값 이월(carry-forward), 폴백은 뷰/상세에서 처리(010/020).
- 신규 상장/거래정지/상장폐지 종목: 마스터 상태에 따라 수집 대상 조정.
- 중복 실행/지연 실행: 멱등 적재(동일 시각 중복 방지).
- 시세 재배포 약관 준수 범위 내 저장/노출.

## 027. 재무/공시/기업정보 수집 배치

- **개요**: 전 종목의 재무제표·공시·기업정보를 1일 1회 수집한다(국내 OpenDART, 미국 SEC EDGAR).
- **액터/권한**: System.

### 입력(트리거·소스)
- 일 1회 스케줄, OpenDART(국내: 재무/공시/기업정보/주식총수), SEC EDGAR(미국: 재무/공시/상장주식수, User-Agent 필수), 대상=전 종목.

### 처리
1. 대상 분할: OpenDART 다중회사 조회(100사 묶음)·전체 매핑 API 우선 활용, 일일 한도 초과 시 분할·이월.
2. 국내 분기 매출 산출: 누적치 차감(2Q=반기−1Q, 4Q=연간−3Q누적).

4. 공시 저장(최신순, 원문 링크), 기업 정형 정보 갱신, 상장주식수(기준일 포함) 분기 갱신.
5. 상태/실패/이월 모니터링 기록.

### 출력(적재·사이드이펙트)
- 분기 재무·공시·기업정보·상장주식수 적재, 매출 태그 미매핑 제외 기업 수 기록, 최종 수집 시각 갱신.

### 엣지케이스
- 일일 한도 초과: 분할·이월(다음 실행 지속).
- 2015 이전 재무: 수집/제공 제외(최소 시작 시점).
- 태그 미매핑/폴백 실패: 매출 집계 제외 + 카운트.
- 신규 상장/재무 미공시 기업: 결측 허용.

- 외부 스키마 변경/장애/개별 종목 실패: 종목 단위 3회 지수 백오프 재시도, 최종 실패는 실패 기록 후 다음 정기 주기에 자동 재포함.

## 029. 일별 체인 지표 사전 집계 배치

- **개요**: 밸류체인별 일별 지표(가치총액/매출 합계)를 스냅샷 구성 기준으로 사전 집계·저장한다.
- **액터/권한**: System.

### 입력(트리거·소스)
- 시세/재무/환율 확정 이후 스케줄, 각 체인의 유효 스냅샷 구성, 일별 종가·상장주식수·분기 매출·환율.

### 처리
1. 각 체인의 해당 일자 유효 구성(스냅샷) 확인.
2. 가치총액 = 구성 상장기업 시가총액 합산(종가 × 최신 상장주식수), 일별 환율로 KRW 환산.
3. 매출 합계 = 구성 상장기업 분기 매출 합산(분기 말일 환율 환산), 태그 미매핑 제외.
4. 커버리지(반영 n/전체 m) 계산. 시세/환율 결측 일자는 직전 관측값 이월(carry-forward)로 집계.
5. 일별 지표 저장. **구조 변경 시 과거 집계는 재계산하지 않고 변경 시점 이후부터 새 구성으로 집계**. 단, **과거 재무/시세/환율 데이터 정정 시에는 영향 기간의 일별 집계를 재계산**한다(구조 변경 재계산 금지와 별개).

### 출력(적재·사이드이펙트)
- 체인별 일별 지표·커버리지 적재(뷰 대시보드 010/타임라인 012의 소스).

### 엣지케이스
- 상장기업 노드 0개: 지표 미산출/커버리지 0.
- 시세/환율 결측 일자: 직전 관측값 이월(carry-forward)로 집계, 차트는 거래일만 표시.
- 구조 변경 당일 경계: 유효 시점(승인/편집 시각) 기준 구성 적용.
- 상장주식수 기준일 과거: 최신 보관값 사용 + 기준일 주석 데이터.
- 데이터 정정(과거 재무/시세/환율): 영향 기간 재계산. 구조 변경분은 재계산하지 않음.
- 대량 체인/노드: 집계 성능·분산 처리.

## 030. LLM 공시 분석·관계 변경안 생성 배치

- **개요**: 신규 공시를 LLM이 분석해 공식 체인의 기존 노드 간 관계 변경안을 생성하고 어드민 검토 큐에 적재한다.
- **액터/권한**: System.

### 입력(트리거·소스)
- 신규 공시(027 수집분) 트리거, 대상=공식 체인, LLM(공급자 미정), 기존 노드 집합·관계 종류 마스터.

### 처리
1. **공시 수집 배치(027) 직후 연쇄 실행**으로 트리거. 신규 공시 중 공식 체인 관련 대상 선별.
2. LLM 분석 → 기존 노드 간 관계 추가/변경/삭제 제안 생성(신규 노드 제안은 범위 밖, 필터). **일일 분석 건수 상한(상수)**으로 비용 제어, 초과분은 다음 실행 이월.
3. 제안에 근거 공시·대상 노드/관계·제안 유형을 부착.
4. 활성 관계 종류로 매핑 가능한 제안만 검토 큐에 대기 적재.

### 출력(적재·사이드이펙트)
- 어드민 검토 큐(022)에 대기 제안 적재. 상태/실패 모니터링 기록.

### 엣지케이스
- 신규 노드가 필요한 관계: 범위 밖 → 배제.
- 제안 대상 노드가 현재 체인에 없음: 무효/보류.
- 비활성 관계 종류로 매핑: 배제/대체.
- LLM 응답 파싱 실패/환각: 검증·필터, 실패 로그.
- 중복 제안: 병합/중복 방지.
- 사용자 체인: 대상 아님(공식 전용).
- LLM 비용/레이트 리밋·타임아웃: 일일 분석 건수 상한(상수)으로 제어, 초과·실패분 다음 실행 이월.
- 실행 주기: 공시 수집 배치(027) 직후 연쇄 실행. LLM 공급자는 기술스택 단계에서 결정.

## 031. 최초 전 종목 과거 데이터 백필 배치

- **개요**: 최초 적재 시 전 종목의 과거 일봉·분기 재무를 일일 한도 내에서 분산 수집한다.
- **액터/권한**: System.

### 입력(트리거·소스)
- 최초 1회(또는 재백필) 트리거, 대상=전 종목, 소스=OpenDART/SEC EDGAR/토스증권, 시작 범위(재무 2015 사업연도 이후).

### 처리
1. 백필 작업 큐 생성(종목·기간 단위 분할), 진행 상태·커서 관리.
2. 일일 API 한도 내에서 분산 실행, 초과분은 다음 실행으로 이월(재개 가능한 체크포인트).
3. 과거 일봉·분기 재무 정규화(027/026과 동일 규칙: 국내 누적 차감, 미국 태그 폴백·Q4 차감) 후 적재.
4. 백필 완료 후 일별 지표 사전 집계(029)로 연결.

### 출력(적재·사이드이펙트)
- 전 종목 과거 시세·재무 적재, 진행률·완료 상태 기록(모니터링 023).

### 엣지케이스
- 중단 후 재개: 체크포인트/커서 기반 이어받기(중복 적재 방지, 멱등).
- 일부 종목 데이터 부재(신규 상장/2015 이전): 가용 범위만 적재.
- 한도 반복 초과: 장기 분산(며칠에 걸쳐 완료).
- 백필과 정기 수집 중복 실행: 충돌 방지(락/우선순위).
- 부분 실패 종목: 종목 단위 3회 지수 백오프 재시도, 최종 실패분은 실패 기록 후 다음 백필 실행에 이월.
- 백필 미완료 상태의 지표 노출: 커버리지/부분 데이터 표기.

> 7. **검색 정렬**: 정확 일치 > 접두 일치 > 부분 일치, 페이지당 20건 + 더보기(상수). (008 반영)
> 8. **자유 주체 노드 필드**: 이름, 주체 유형(소비자/정부/비상장기업/기타), 설명 메모(선택) 3개. (011·015 반영)
> 9. **관계 방향성·중복**: 마스터에 방향성 속성(유향/무향) 정의. 동일 노드 쌍 같은 종류 중복 불허, 다른 종류 병존 허용, 자기 참조 불허. (016 반영)
> 10. **노드 중복**: 한 체인에 동일 종목 노드는 1개만. (015 반영)
> 11. **체인 이름 중복**: 동일 사용자 내 불허, 전역(타 사용자 간) 허용, 공식 체인은 전역 유일. (013·014·018 반영)
> 12. **편집 초안**: 자동 저장 없음, 미저장 이탈 시 경고 다이얼로그만(MVP). (013·018 반영)
> 13. **사용자 체인 타임라인**: 지원. 저장마다 스냅샷 기록, 사용자 체인 뷰에서도 복원 가능. (012·018 및 C·D 그룹 스냅샷 서술 일괄 반영)
> 14. **스냅샷 생성 단위**: 편집 저장 1회당 1스냅샷(공식/사용자 동일), LLM 승인 1건당 1스냅샷, 동시 발생은 시각 순 직렬 처리. (018·021·022 반영)
> 15. **공식 체인 삭제**: 물리 삭제 금지, 보관(비공개 전환). 복제본·스냅샷 영향 없음. (021 반영)
> 16. **관계 종류 이름 변경**: ID 참조 유지, 표시 라벨은 최신 이름을 따름(라벨 이력 미보존). (024 반영)
> 17. **결측 처리**: 시세/환율 결측 일자는 직전 관측값 이월(carry-forward), 차트는 거래일만 표시. (용어·전제, 010·012·026·028·029 반영)
> 18. **재집계**: 데이터 정정 시 영향 기간 일별 집계 재계산, 구조 변경분은 재계산 안 함. (027·029 반영)
> 19. **LLM 배치**: 공시 수집 배치 직후 연쇄 실행, 일일 분석 건수 상한(상수). 공급자는 기술스택 단계 결정. (030 반영)
> 20. **실패 재시도**: 종목 단위 3회 지수 백오프, 최종 실패는 기록 후 다음 정기 주기 재포함, 백필 실패분은 다음 백필 실행 이월. (026·027·031 반영)
